# 实测 · Mac 上 LLM 的"极速天花板"与三值加速曲线（2026-05-30）

> Michael 让我"跑吧"——真在 Mac 上跑真模型，拿到真 tok/s + 看质量；并找"能在 M4 Max 128G 上极速(几万 tok/s)跑起来的最大模型"。
> ⚠️ **这次跑在主力 mac（base Apple M4 / Mac16,1 / 24GB / 4P+6E, 带宽 ~120GB/s），不是 aux M4 Max 128GB（带宽 ~410–546GB/s, ~3.5–4×）。** 曲线形状 + 小端天花板可直接迁移；大端(带宽瓶颈)数字在 M4 Max 上 ×~3.5–4。
> 引擎：mainline llama.cpp 9430（brew 装，原生支持 i2_s + 三值 TQ1_0/TQ2_0）。原始命令全部 `llama-bench`/`llama-completion`，模型放在本地 models 目录。

## 一、极速天花板曲线（不同尺寸，取每点最优配置）

| 模型 | 参数 | 字节 | 最优 tok/s | 后端 | 状态/瓶颈 |
|---|---|---:|---:|---|---|
| stories260K | 0.29M | 1.1 MB | **~10,670** | CPU 2–4线程 | 整个住进缓存 → 不搬数据 → 卡在**框架/时钟开销** |
| stories15M (q8) | 24M | 25 MB | ~2,460 | CPU | 缓存边缘，过渡区 |
| stories15M (f16) | 24M | 93 MB | ~957 | CPU | ~SLC 边缘 |
| TinyLlama-1.1B (TQ1_0 三值) | 1.1B | 282 MB | ~102 (CPU) | CPU | 超缓存 → DRAM 流式 |
| TinyLlama-1.1B (q4) | 1.1B | 606 MB | ~113 (Metal) / 79 (CPU) | Metal | **带宽瓶颈** |
| TinyLlama-1.1B (q8) | 1.1B | 1090 MB | ~71 (Metal) | Metal | 带宽瓶颈 |
| TinyLlama-1.1B (f16) | 1.1B | 2050 MB | ~43 (Metal) / 26 (CPU) | Metal | 带宽瓶颈 |

**关键转折：小模型 CPU 比 Metal 快（0.29M: CPU 10,670 vs Metal 1,731）——GPU 启动开销在小模型上压倒一切；大模型 Metal 比 CPU 快（1.1B-q4: Metal 113 vs CPU 79）——超缓存后拼内存带宽，Metal 带宽高。**

## 二、回答 Michael 的问题

1. **"几万 tok/s 的最大模型"**：在 llama.cpp 上，**万级(~10k+)天花板 ≈ 0.3M 参数**（stories260K 实测 ~10,670 t/s，最优 2–4 性能核；8 线程反而掉到 ~4.6k，是能效核的线程同步拖累）。要冲**几万(20k+)**：要么 <0.1M 参数，要么换**裸金属 runtime**（microGPT 4k 参数裸跑可达百万级——见 Michael 引的 talos-vs-macbook：单 M4 核比 FPGA 的 53k 还快 ~71×）。
2. **为什么有天花板**：`tok/s ≈ 带宽 ÷ 每字节搬运量`。模型小到能**整个住进缓存**（≤ 几 MB）→ 不搬权重 → 卡在框架/时钟开销（千~万级）；一旦超缓存（≥~100MB）→ 每个字都从 DRAM 搬 → **带宽瓶颈** → 跌到几十~低百级。
3. **生成 vs 并行**：stories260K 的 `pp512`(并行处理)=22,632 t/s，`tg`(逐字生成)=~11k——证明**生成被"逐字串行的每-token固定开销"卡住，不是算力**。这正是定制芯片(去掉框架开销+片上常驻)能赢的地方。

## 三、三值的真实测量（同一个 1.1B，5 种格式，CPU 公平对比）

| 格式 | bpw | 字节 | tok/s | 备注 |
|---|---:|---:|---:|---|
| **TQ1_0 三值** | 1.69 | **282 MB** | **101.9** | 字节最小 → 最快 ✓ |
| TQ2_0 三值 | 2.06 | 325 MB | 57.9 | **比 q4 还慢**（内核没调优）⚠️ |
| Q4_0 | 4.5 | 606 MB | 79.3 | NEON 高度优化 |
| Q8_0 | 8.5 | 1090 MB | 53.1 | |
| F16 | 16 | 2050 MB | 25.9 | |

**三个诚实结论：**
- **三值真的最小最快**（TQ1_0 282MB / 101.9 t/s，全场第一）。f16→q8→q4 严格"越小越快"（26→53→79），印证带宽律。
- **但 TQ2_0 比 q4 慢**（57.9 < 79.3，尽管字节更少）——纯软件内核没为三值调优。**这恰恰是定制三值芯片数据通路要消灭的"软件税"**：硬件原生三值（加/减/跳零，无乘法器）能拿满它的理论优势。
- **质量警告（实测）**：事后把普通模型三值化 → 输出垃圾（"ispertypertyperty…"）。**三值近无损必须"训练时就三值"（BitNet 2B4T，文献 ~FP16 水平）**，不是事后量化。→ 速度优势随便拿；质量优势要训练。

## 四、踩过的坑（存档，省得重踩）
- microsoft/BitNet-2B-4T 的 `ggml-model-i2_s.gguf`（modelscope/HF 唯一文件）用了已被 mainline llama.cpp **移除的 `IQ4_NL_4_4`(type 36)** 打包 → 只有老 bitnet.cpp fork 能加载；而 fork 对 2B-4T 用的是 3B 的 codegen 内核参数 → 矩阵分块不匹配 → 输出全 `@`。两边都跑不对**这个文件**。
- 根因链：所有"转成干净 GGUF"的路都被 **py3.13 + torch~=2.2.1(py3.9) 冲突**卡住（gguf-py 装不上）。绕过办法：mainline llama.cpp 自带 `llama-quantize` 支持 TQ1_0/TQ2_0，**直接从 f16 量化出三值**，不需要 torch。
- 三值 TQ1_0/TQ2_0 在 llama.cpp **没有 Metal 内核**（`-ngl 99` 会 SIGABRT 134）→ 只能 CPU 跑。

## 五、对芯片论点的意义（这条曲线就是芯片论证的"尺寸轴"实证）
- "塞进片上 → 飞快 / 流式 DRAM → 慢" 不再是 slide，是**主力 M4 上量出来的**：1.1B 同模型 6.4× 字节差 → 4.4× 速度差（f16 26 → TQ1_0 102，按各自最优）。
- **三值把模型左移**（1.1B: 2050MB→282MB），往"住进缓存的快区"推。M4 Max 128GB 上：大端 ×~3.5–4，且能装下 30–70B 三值（128GB 内存）；一个**训练时三值**且小到能片上常驻的模型，会直接跳进万级区。
- **TQ2_0 的软件税 = 定制芯片的机会**：硬件三值通路把那部分"白送"回来。

## 六、M4 MAX 128GB 实测（2026-05-31，aux 机，回答 Michael 原问题）
> 跨机 SSH 驱动 aux（M4 Max, 12P+4E, ~410GB/s）。⚠️ CPU 段被 aux 上 Michael 的 python 任务争用（部分 ±大），**Metal/GPU 段干净**（GPU 不被 CPU 任务争）。

**同一个 1.1B 模型 · Metal · main M4 vs M4 Max（干净，钱景对比）:**
| 格式 | 字节 | main M4 (~120GB/s) | **M4 Max (~410GB/s)** | 倍数 |
|---|---:|---:|---:|---:|
| q4_0 | 606MB | 113 | **313** | 2.8× |
| q8_0 | 1090MB | 71 | **164** | 2.3× |
| f16 | 2050MB | 43 | **110** | 2.6× |

有效带宽（M4 Max Metal）≈ 180–226 GB/s（q4 313×606=190、f16 110×2050=226）vs main M4 ~70–90 GB/s → **比值 ~2.5× = 带宽比**，确认大端是带宽瓶颈。
**小端天花板（stories260K, CPU）:M4 Max 峰值 11,629 t/s(4线程) ≈ main M4 10,670** → **万级天花板与机器无关(开销/时钟封顶);带宽只帮大端。**

**结论(回答"M4 Max 128G 能极速跑多大"):**
- 极速(万级)天花板 ~0.3M 参数,**M4 Max 不比 base M4 快**(同 ~1.1万,框架开销主导)。要更快靠裸金属(TALOS-mac:micro 18万)。
- 大模型 M4 Max 比 base M4 快 ~2.5×(带宽);且 128GB 能装 base 24GB 装不下的(30–70B 三值)。
- 三值 CPU 段(争用,指示性):TQ2_0 183、q4 157、f16 40 —— 三值最快但内核噪声大,印证"软件三值税"。
