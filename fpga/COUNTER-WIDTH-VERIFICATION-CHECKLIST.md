# 读出位宽 / 计数器宽度 — RTL 验证检查清单
*起因:CIM-Explorer 模拟器压测(2026-06-08)发现:64 行三值累加,读出 ≥6 bit 才无损;单个 5-bit 读出会截断 → 精度 74%→55%。本清单把这条变成可在 RTL/Tile 设计里逐项核对的规则。*

> 证据脚本:`01-chip-core/_reference/tools/cim-explorer-STRESS-deep-ternary.py`([B] resolution sweep)。

---

## 0. 先分清两条路（关键，别混）
| 路径 | 是否会截断 | 现状 |
|---|---|---|
| **数字加法树**(本目录 `ternary_mac_array.v`,FPGA/数字 shuttle M0/M1) | **否,天生无损** | ✅ `SUMW = AW + 1 + ceil(log2 P)`,逐级长 1 bit,正好够 P 项求和的最坏情况 |
| **模拟/计数器读出**(ReRAM CIM 宏,SPIKA 式 5-bit 上下计数器,M2) | **会,若计数器盖的行数超出量程** | ⚠️ 模拟器压测命中的就是这条 |

→ **模拟器抓到的"5-bit 不够"是针对 M2 的 ReRAM 计数器读出,不是我们 FPGA 的数字加法树。** 数字加法树已无损,本清单第 1 节确认它、第 2 节才是真正要补的 M2 规则。

---

## 1. 数字加法树(`ternary_mac_array.v`)— 确认无损（应已满足）
- [ ] **位宽公式**:`SUMW == AW + 1 + LOG2P` 且 `LOG2P == ceil(log2(P))`。（P=64,AW=8 → SUMW=15）
- [ ] **逐级位增长**:加法树每级节点宽度 +1,最终 `dot` 宽 = SUMW,**中途无截断/饱和**。
- [ ] **下游不收窄**:`dot`(SUMW)送进下一级/累加器时,目标寄存器 ≥ SUMW,无隐式截位。
- [ ] **测试向量**:对 P=64/256/1024,喂"全 +1 权重 × 满幅输入"极端向量,比对加法树输出 == numpy 精确点积(**位精确**)。

## 2. ReRAM 计数器读出(M2 宏)— 新增必核规则
**规则 R1（量程）**:一个 B-bit 有符号上下计数器只能表示 ±(2^(B-1)−1)。
- [ ] 设 `R_tile` = 馈入**单个计数器**的最大有效(非零)三值行数。要求 **`R_tile ≤ 2^(B-1) − 1`**(无饱和)。SPIKA 5-bit → 单计数器最多安全盖 **≤15** 行(有符号),或按实际编码 ≤31。
- [ ] 若工程上想让一个计数器盖更多行(如 SPIKA 用 5-bit 盖 64 行=**主动量化**),必须:
  - [ ] 明确这是**有损量化**,并用模拟器复测该 tile 尺寸下的端到端精度(见第 3 节),拿到"可接受"的书面结论;
  - [ ] 在 datasheet/RTL 注释里标注饱和行为(钳位 or 回绕)。

**规则 R2（分块累加,推荐路径）**:把长累加 K 切成 tile,每 tile 一个计数器,**跨 tile 用更宽的数字累加器无损相加**。
- [ ] 选 `R_tile` 满足 R1;tile 数 `= ceil(K / R_tile)`。
- [ ] **跨 tile 数字累加器宽度** `≥ ceil(log2(K)) + 1`(有符号,覆盖 −K…+K)。K=4096 → ≥13 bit。
- [ ] 跨 tile 求和**全精度、无截断**(这一段就是数字加法树,复用第 1 节保证)。
- [ ] 差分 ± 列各自计数,最后**数字域相减**(SPIKA:相减在计数器内完成),核对符号/借位正确。

**规则 R3（LLM 长累加,我们的真实负载）**:transformer 的 K 可达数千 → **绝不能靠单个 5-bit 计数器盖全 K**。
- [ ] 确认 tiling 把每计数器的 `R_tile` 压到 R1 量程内;
- [ ] 确认有跨 tile 数字累加器(R2),且其位宽按最大 K 配。

## 3. 闭环复测（用模拟器把 RTL 决策验回去）
- [ ] 在 `cim-explorer-STRESS-deep-ternary.py` 里把 `resolution` 设成**所选计数器位宽**、矩阵 N 设成**所选 R_tile**,跑 [B] sweep,确认该点精度无塌（对比 resolution=20 的无损基线掉幅在可接受范围）。
- [ ] 把器件噪声(LRS/HRS_NOISE)设成 **M2 回片实测分布**,复跑,出"该 tile 设计下的预测精度"。

## 4. 一句话验收
> **数字加法树已无损(SUMW 公式);M2 计数器读出必须满足 `R_tile ≤ 2^(B-1)−1` + 跨 tile 全精度数字累加(≥⌈log2 K⌉+1 bit)。任何"单计数器盖超量程行数"都必须当有损量化、用模拟器复测签字。** 这条是模拟器在流片前替我们抓到的,省的就是 ¥50 万回片才发现精度塌。
