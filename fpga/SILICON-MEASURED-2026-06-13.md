# 我们的三值阵列在真芯片上算对了 — 硅上实测 (2026-06-13)
*Our ternary MAC array runs correct on real silicon*

---

## 说人话：发生了什么、意味着什么（先读这段）

**一句话**：我们自己设计的那块"**不用乘法器**"的 AI 计算电路，第一次在一块 **¥150 的真芯片**上跑起来了，而且**算得分毫不差**。

**这意味着什么（这才是重点）：**
- 以前我们只能在**仿真里**说"这个电路是对的、喂数喂得饱"。今天，这两件事在**真硅上**被证明了——芯片可行性证据里**最硬的一档**。
- 它直接干掉了别人最容易质疑的两个点：①"你这阵列在真硬件上真能算对吗？"——**能，逐比特对**。②"数据喂得进去、不会卡住吗？"——**喂得满，一拍一个结果不停顿**。
- 它**不是**"芯片造好了/流片了"。这是一块学习用的 FPGA 开发板，**用来验证核心电路逻辑**；产品级芯片的速度数字来自物理模型投影（见下），**和这块板子严格分开**。

> **一句话边界**：真硅上 measured 的是"**电路算对 + 喂数撑住 + 没用乘法器**"；产品级 tok/s 是 **projected**（物理模型，下面单独标）。两者绝不混。

---

## 实测数字（MEASURED，真芯片 xc7z010 / EBAZ4205，读寄存器读出来的）

读法：芯片跑完后，用 Linux 直接读它的寄存器（`/dev/mem` + AXI），**不是看灯、不是仿真**。

| 测的什么（说人话） | 实测值 | 怎么证的 |
|---|---|---|
| **算得对不对** | 片上校验和 `0x3cc57dcf` == 仿真标准答案，**逐比特一致**（4096 个输出全对） | AXI 读寄存器，两次稳定 |
| **喂数卡不卡** | II=1：4096 个点积 / **4102 拍** == 理想 4102（基本一拍一个，不停顿） | 寄存器读周期计数 |
| **用没用乘法器** | **0 DSP**（结构上无乘法器 = 三值命题的硬证据） | 综合利用率报告 |
| **跑多快（这块板）** | 布线后 **136 MHz**；P=64 → 3.2 GMAC/s | 时序报告 / 寄存器 |
| 资源占用 | LUT 1538 (8.7%) / FF 1827 (5.2%) / BRAM 9.5 / **DSP 0** | 利用率报告 |

> 这块板本身跑大模型只有 **<1 tok/s**——它的价值是**验证核心电路**，不是当产品。

## 产品级速度（PROJECTED — 物理模型投影，严格区别于上面的 measured）

把硅上量到的"算对 + 喂数满"放进我们的三轴物理模型（算力 / 内存搬运 / 功耗，**真实利用率只取 0.12**、内存搬运已计入），投影到产品级 28nm 存算一体芯片跑 9B 模型：

| 情形 | sustained tok/s | 说人话 |
|---|---|---|
| CONS（保守） | **~1,200** | 最差也比消费级 GPU(60)/M4 Max(80) 快一个数量级 |
| BASE（中性，含投机解码） | **~8,300** | 落在 Groq(2,100) 之上、d-Matrix 之下 |
| OPT（乐观） | **~49,000** | 瓶颈转到内存搬运（印证"最终都是内存墙问题"） |

> **诚实锚点**：不带投机解码的**裸 baseline ≈ 2,091 tok/s ≈ Groq LPU 实测**——自洽，不是吹。`util` 是最软的参数，我们摆明了它、欢迎红队打。

## 护城河是什么（说人话）

不比"峰值算力(TOPS)"那个数——那是数据中心的战争、迟早被商品化。我们比三样物理上的东西：
1. **每瓦能跑多少 token**（无乘法器 → 省电）；
2. **能不能绕开内存墙**（三值权重比 FP16 小 ~8× → 搬得少 → 单流更快）；
3. **确定性 + 端上私有**（断网照跑、数据不出设备、不可被远程关掉）。

---

## 诚实边界 / 还没证的（keep honest）

- **没流片**：这是 FPGA 验证，不是 ASIC。产品芯片还在设计阶段。
- **只测了 P=64**：阵列放大到 P=512 是综合可扩（194–266MHz 综合估计），但**还没在硅上跑过 P=512**。
- **只测了乘加数据通路（GEMM）**：attention / norm / KV 那些"气泡"没单独测——它们体现在投影里的"真实利用率 0.12"。
- **板上精确 tok/s 还没读出**：这块板给的是"对/不对 + II=1"，不是一个 tok/s 数字（下一版加片上计数器经 UART 读真实频率）。

## 一个真实的工程教训（doers not talkers）

这次硅上验证不是一帆风顺：板上的指示灯一连 6 次报"红"（看起来像算错了），我们差点信了。最后**直接去读芯片里那个 32 位寄存器**，才发现芯片**一直算的是对的**——是那颗**指示灯接反了**在骗人。

教训一句话：**信直接测量，别信间接指标。** 仿真能过、综合能过、灯还可能骗你——只有把真实的数读出来，才算数。这也是为什么这份文档里每个数字都标了它是 measured 还是 projected。
