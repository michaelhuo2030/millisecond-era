## 说人话：发生了什么、意味着什么（先读这段）

**一句话**：我们自己设计的那块"**不用乘法器**"的 AI 计算电路，第一次在一块 **¥150 的真芯片**上跑起来了，而且**算得分毫不差**。

**这意味着什么（这才是重点）：**
- 以前我们只能在**仿真里**说"这个电路是对的、喂数喂得饱"。今天，这两件事在**真硅上**被证明了——芯片可行性证据里**最硬的一档**。
- 它直接干掉了别人最容易质疑的两个点：①"你这阵列在真硬件上真能算对吗？"——**能，逐比特对**。②"数据喂得进去、不会卡住吗？"——**喂得满，一拍一个结果不停顿**。
- 它**不是**"芯片造好了/流片了"。这是一块学习用的 FPGA 开发板，**用来验证核心电路逻辑**；产品级芯片的速度数字来自物理模型投影（见下），**和这块板子严格分开**。

> **一句话边界**：真硅上 measured 的是"**电路算对 + 喂数撑住 + 没用乘法器**"；产品级 tok/s 是 **projected**（物理模型，下面单独标）。两者绝不混。

## 实测数字（MEASURED，真芯片 xc7z010 / EBAZ4205，读寄存器读出来的）

读法：芯片跑完后，用 Linux 直接读它的寄存器（`/dev/mem` + AXI），**不是看灯、不是仿真**。

| 测的什么（说人话） | 实测值 | 怎么证的 |
|---|---|---|
| **算得对不对** | 片上校验和 `0x3cc57dcf` == 仿真标准答案，**逐比特一致**（4096 个输出全对） | AXI 读寄存器，两次稳定 |
| **喂数卡不卡** | II=1：4096 个点积 / **4102 拍** == 理想 4102（基本一拍一个，不停顿） | 寄存器读周期计数 |
| **用没用乘法器** | **0 DSP**（结构上无乘法器 = 三值命题的硬证据） | 综合利用率报告 |
| **跑多快（这块板）** | 布线后 **136 MHz**；P=64 → 3.2 GMAC/s | 时序报告 / 寄存器 |
| 资源占用 | LUT 1538 (8.7%) / FF 1827 (5.2%) / BRAM 9.5 / **DSP 0** | 利用率报告 |

> 这块板本身跑大模型只有 **<1 tok/s**——它的价值是**验证核心电路**，不是当产品。

## 护城河是什么（说人话）

不比"峰值算力(TOPS)"那个数——那是数据中心的战争、迟早被商品化。我们比三样物理上的东西：
1. **每瓦能跑多少 token**（无乘法器 → 省电）；
2. **能不能绕开内存墙**（三值权重比 FP16 小 ~8× → 搬得少 → 单流更快）；
3. **确定性 + 端上私有**（断网照跑、数据不出设备、不可被远程关掉）。

## 抗噪演示：拨一下开关，板子从红变绿（EMULATED · 2026-06-14 新增）

### 在测什么

把一个训练好的手写数字分类器烤进 FPGA。板子上**同时跑两条路**：

- **三值路**：权重是 {-1, 0, +1}，映射到电导范围 [-S, S]
- **INT8 路**：权重是 8-bit 整数，映射到**同一个** [-S, S] 范围

然后注入**同样强度的随机噪声**，模拟 ReRAM 电导漂移。

板子对 **16 张固定测试图片**各做一次分类，统计各自答对几张。

### 结果（devmem 读出，与仿真逐比特一致）— 全 5 点 MEASURED 2026-06-15

| 噪声强度 A | 三值答对 | INT8 答对 | LED | 原始寄存器 |
|---|---|---|---|---|
| A = 0（无噪声） | 15 / 16 | 16 / 16 | 🔴 红（INT8 略好） | 0xbeef0f10 |
| **A = 32** | **15 / 16** | **14 / 16** | 🟢 绿（三值赢） | 0xbeef0f0e |
| A = 64 | 14 / 16 | 10 / 16 | 🟢 绿 | 0xbeef0e0a |
| A = 96 | 11 / 16 | 10 / 16 | 🟢 绿 | 0xbeef0b0a |
| A = 128 | 10 / 16 | 7 / 16 | 🟢 绿 | 0xbeef0a07 |

**临界点 A* ≈ 16–32**：加一点噪声，三值就赢了，而且越噪越强。A=128 时三值领先 3 分（10 vs 7）。

**拨噪声开关，板子从红变绿。**

### 为什么三值在噪声下赢

INT8 把权重塞进 256 级，相邻级之间间距极窄。一点噪声就把数值推过决策边界，分类结果翻转。三值只有三个值（-1、0、+1），决策间距是 INT8 的几十倍——同样的噪声推不过去，答案稳住。

这正是 ReRAM 器件的物理困境：电导在制造和使用中会漂移，精度越高的映射越脆弱。三值是把决策间距做到最宽的极限。

### 诚实边界

- FPGA 是数字芯片，**没有真实器件噪声**——这是忠实仿真 ReRAM 电导噪声（同范围映射 + 同绝对噪声量），不是"测到了 FPGA 的器件噪声"
- 物理本身（σ=0.10，三值 0.89 vs INT8 0.67）已由 Python 仿真 measured；FPGA 是把这套物理做成可以拍视频的硬件演示
- A=0/32/64/96/128 全 5 点已在硅上测量 ✅（2026-06-15 完成）

## Model-S 三值 LLM：BRAM 部署可行性（PROJECTED，2026-06-15）

第一个真正训练出来的三值 LLM — 在 M4 Max 上用 MLX 完成 QAT，现在计算能不能塞进这块 ¥150 的板子。

**模型参数（实测训练）：**
| 项目 | 数值 |
|---|---|
| 架构 | 8层, d=64, vocab=256, char-level |
| 总参数 | 558,144 |
| 三值参数（QAT后） | 524,288 |
| 稀疏度（零值） | 31.7%（+1: 34.2%，−1: 34.1%） |
| 训练 val loss | **2.4609**（基线 5.545，真收敛） |
| BRAM 需求（2bit打包） | **128 KB** |

**EBAZ4205 BRAM 预算对比：**
- 总容量：270 KB（60×BRAM36）
- 可用（80%利用率，扣阵列开销）：184 KB
- Model-S 占用：128 KB → **剩余 56 KB**（可放 KV cache 或第二模型）
- Model-M（14层）估算：180 KB → 勉强装得下

**tok/s 推算（PROJECTED，基于 P=512 硅上实测 II=1）：**
| 项目 | 数值 |
|---|---|
| 每 token MACs（三值 GEMM） | 524,288 |
| P=512 阵列，100 MHz | 51.2 GMAC/s |
| 理论最优周期数 | 1,024 cycles / token |

| **倍数优势** | **400× faster（PROJECTED）** |
| 权重全在 BRAM，无 DDR 访问 | 0 内存墙 |

## 诚实边界 / 还没证的（keep honest）

- **没流片**：这是 FPGA 验证，不是 ASIC。产品芯片还在设计阶段。
- **P=512 已逐比特验证**：校验和 `0x6b78249f` == 仿真 golden，II=1，0 DSP，75.5% LUT（2026-06-14 升级）。
- **只测了乘加数据通路（GEMM）**：attention / norm / KV 那些"气泡"没单独测——它们体现在投影里的"真实利用率 0.12"。
- **板上精确 tok/s 还没读出**：这块板给的是"对/不对 + II=1"，不是一个 tok/s 数字（下一版加片上计数器经 UART 读真实频率）。

## 一个真实的工程教训（doers not talkers）

这次硅上验证不是一帆风顺：板上的指示灯一连 6 次报"红"（看起来像算错了），我们差点信了。最后**直接去读芯片里那个 32 位寄存器**，才发现芯片**一直算的是对的**——是那颗**指示灯接反了**在骗人。

教训一句话：**信直接测量，别信间接指标。** 仿真能过、综合能过、灯还可能骗你——只有把真实的数读出来，才算数。这也是为什么这份文档里每个数字都标了它是 measured 还是 projected。
