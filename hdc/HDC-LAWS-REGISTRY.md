# HDC LAWS REGISTRY — the locked, must-use laws (卡死)

> **Purpose**: stop re-discovering what we already know. Every law we've *earned* lives here, with its **provenance** (which experiment proved it) and its **enforcement** (how it becomes a default / gate so future work CAN'T ignore it).
>
> **The discipline (Michael, 2026-05-29):** "We find good laws, then the next experiment doesn't use them — it falls back to the LLM's generic knowledge. Lock them in (卡死)." A law in a doc you must *remember* to read is passive and gets lost. A law baked into the experiment kit's **defaults** and **gates** is active and can't be skipped. This registry is the single source of truth; the kit (`hdc_experiment_kit`, spec'd separately) enforces it in code.
>
> **Rule: before any HDC experiment, the kit loads this registry. No experiment emits a verdict that violates a GATE law. No experiment picks an encoding/param without consulting a DEFAULT law (or explicitly overriding with a reason).**

## TIER 0 — META-LAW (governs HOW you enter the HDC space, above the algebra laws)
**L0. Transforms are lenses on HDC's possibility space.** The encoder/operator R is a *map* input→high-D; every mathematical transform (dense-Gaussian, Hadamard/Fastfood, Fourier/phasor, wavelet, circulant, permutation, log, learned) is a different **warp** that changes **cost + inductive bias + interpretability + preserved geometry** — but NOT fundamental superposition capacity (D/(2 ln N), code-independent; see L1). HDC's expressive limit is thus not a number but a **family of surfaces** spanned by (transform × task-structure); exploring HDC = mapping that family with the ruler. **Win = match the warp to the structure (道法自然).** **Measured refinement (2026-06-01, lens-armory):** the lens family is GATED by *dense mixing* — a usable lens must spread input energy across dims before sign-encoding. Dense-Gaussian / Hadamard-Fastfood / circulant / Toeplitz all mix densely and tie (the cheap circular wrap is free); a hierarchically-LOCALIZED orthonormal transform (Haar wavelet) DETONATES (+360–610% PPL) because it concentrates energy instead of spreading it. So "match warp to structure" is necessary, but candidate transforms must first pass the dense-mixing gate. (Evidence: `lens_toeplitz` CONFIRM, `lens_haar_wavelet` FALSIFY.)
- **Provenance**: Fastfood = first worked lens (attention +2.2% ≈ dense, O(in·logD)); sparse-R = first failed lens (+632%, too little mixing); Fourier (2× capacity), permutation (sequence) earlier. 2026-06-01. `feedback_transforms_as_lenses_hdc_foundational_2026_06_01`; doc `11-systems-science/HDC-FOUNDATIONAL-LAW-transforms-as-lenses.md`; program `HDC-TRANSFORM-LIBRARY-PROGRAM.md`.
- **Enforcement (DESIGN META-GATE)**: before picking an encoder, ask *"which warp fits this structure, and what does it cost vs. preserve?"* — don't default to dense-Gaussian by habit; consult the transform library. Every new lens earns its place only via the ruler (faithfulness × cost × structure × interpretability).

## TIER 1 — THE BOUNDARY LAW (the master thread)
**L1.** HDC's algebra is a real edge **when structure is GIVEN/CLEAN** (you bind known roles↔fillers; you define the compositional space). It **ties learned embeddings when structure must be DISCOVERED from real messy semantics** (embeddings already captured it; bipolar HDC ≈ lossy binary mean-pooling).
- **Provenance**: W1–W5 wins (given structure) vs Wisdom-Atlas v3 null (discovery), 2026-05-29. `project_hdc_algebra_boundary_law_2026_05_29`.
- **Enforcement (DESIGN GATE)**: before proposing any HDC application, answer *"is the structure given/clean or discovered-from-real-data?"* Discovery → use LLM/embeddings for that step; HDC only for bind/store/serve. **Architecture default: LLM discovers → HDC binds/holds.**

## TIER 2 — RIGOR GATES (an experiment is VOID if it violates one)
**L2. Reconstruction ≠ task.** "X replaces Y" needs an end-to-end metric (perplexity/task acc), not cos/recall. *(Flywheel: cos>0.99 but +17.8% ppl.)*
**L3. Descriptive ≠ causal.** Causal claims need intervention + a control where random does worse. *(Paper#2 ablation KL=0.002.)*
**L4. N=1 is a PoC, not a result.** Golden Standard = ≥4 encodings × ≥3 D × ≥2 sparsity × ≥3 seeds.
**L5. Headline = artifact.** Every number traces to a CSV/JSON. Reconcile the moment data changes. *(Audit artifacts, not status docs — I missed real results 3× by trusting status notes.)*
**L6. Theory-Result Triage.** A null vs strong theory → suspect the tool first (state mechanism → audit design → fix → re-run → only then revise theory). Anti-shield lock: name flaw + predict fix + run it. **A FALSIFY (null) is a CLAIM held to the SAME rigor bar as a CONFIRM** — held-out + baseline + ≥3 seeds, else it downgrades to POC. *(Enforced in kit `_gate` 2026-06-01: it had only checked CONFIRM, letting 8 single-seed/no-held-out legacy results enter as FALSIFY and falsely flag L0 MIXED; fixed + 8 polluters retro-downgraded to POC. An ungrounded "it failed" is as dangerous as an ungrounded "it worked".)*
**L7. Solvability + Engine-match.** A task counts only if (a) some baseline beats random (else RETIRE — not the theory's fault), and (b) it fits HDC's nature (don't test a holographic computer on linear-tabular tasks).
- **Enforcement (RUNTIME GATE)**: the kit refuses to emit CONFIRM/FALSIFY unless: lift-over-baseline reported (not raw acc), bootstrap CI computed, held-out split used, a flat/random baseline included, solvability gate passed. Violations → the verdict is auto-stamped "VOID — gate failed."

## TIER 3 — METHOD DEFAULTS (use these unless you have a logged reason not to)
**L8. Encoding shelf + when to use** (always sweep ≥4, but defaults):
- `bipolar` — dense retrieval workhorse (~98.5% quality); **note: bipolar bundle/cosine ≈ mean-pooling+cosine** (so it won't beat embeddings on discovery — see L1).
- `sparse_ternary` — descriptive monosemanticity / fingerprinting (NOT causal). Best for Paper-#2-style geometric separability.
- `Fourier` — **2× capacity vs bipolar** (D=50K Fourier ≈ D=100K bipolar). Default when capacity-bound. *(paper-bundle-capacity.)*
- `multi-bit` — encoder-dependent gain; sweep, don't assume.
**L9. Readout/calibration for replacement**: **least-squares per-head readout** (flywheel +15.2%); **ridge/shared blows up** (ppl 18–86,000). Per-row FFN calibration mandatory for joint replacement. *(flywheel results_v_ridge_perhead_sweep.csv.)*
**L10. Layer choice is task-dependent**: text-attribution peaks EARLY (Mao L0–L2); concept-classification peaks DEEP (Qwen ~L32–36); DeepSeek-MoE routes SHALLOW (L4). Don't assume a universal "best layer."
**L11. Cross-domain/cross-language needs LLM abstraction FIRST**: surface-encoder HDC fails across vocab/language; abstract via LLM → then HDC bridge. *(universal_concepts; `feedback_cross_domain_requires_llm_abstraction_first`.)*
**L12. Capacity rule**: usable items ≈ D/20 (conservative; real collapse ~2× that). *(C3.)*
**L13. Tooling traps**: use **FP16 model loading** for embedding extraction — `mlx_lm.load()` packing returned 512-dim packed views instead of real 5120-dim, silently corrupting 3+ tests. *(STEELMAN 2026-05-23.)*

## TIER 4 — ENGINEERING DEFAULTS (on by default, not afterthoughts)
**L14.** MLX GPU encode (97–165×), NEON `int8_dot` similarity (39–63×), streaming for D≥500K, checkpoint every >2-min compute. *(`hdc_speed_and_variants` benchmark.)*
**L15. Dispatch**: long runs → Kimi via tight spec (zero judgment calls) → **Claude audits before any "verified" label** (non-negotiable). Every Kimi spec must cite which L# laws it applies.
**L16. int8-overflow gate (卡死, 3rd occurrence)**: NEVER `int8 @ int8` / `np.sum(int8)` on hypervectors — a sum of D ±1 wraps mod 256 → similarity/argmax goes random and silently fakes a NULL (bit it the T1 INT8 readout, the T2 stage-0 cleanup `keymat@probe` "HDC<bag", and the T2-LLM-DAG cleanup "recall 0.15<random"). **Always widen to int32 before any dot/sum over a hypervector** (`keys.astype(np.int32) @ probe.astype(np.int32)`). A too-good NULL is as suspect as a too-good positive (guardrail-5). *(`feedback_lock_in_laws_executable_not_passive`.)*
**L17. MEASURED ≠ EXPECTED (卡死, anti-fabrication)**: a number enters a doc/memory/result ONLY if a saved artifact (JSON/CSV/run-log) confirmed to exist THIS turn backs it. Never write a predicted/expected value in the past tense. When a run crashes, the honest artifact is "crashed — no result", never a plausible placeholder. A fabricated number is MORE dangerous than a crash (it looks true and survives into the knowledge base). *(Incident 2026-06-01: under a mid-session Mac reboot + tangled parallel tool-calls, fabricated contextual-lens AUCs (≈0.79–0.84, "Finding 8 leakage table") were written into 3 docs before the run — which never succeeded — completed; forensics showed the values existed only in my own tool-call text, no log/JSON ever produced them. Purged + re-run for real (0.672/0.677/0.671). `verification/2026-06-01-fabricated-numbers-incident-L5.md`.)* Enforcement: before any Edit that adds a metric, the value must be quotable from a path; "not run / crashed" is a legal disposition, an invented number is not.

It **cannot DISCOVER the law from data** (L1: discovery favors learned encoders) and **cannot beat computational irreducibility** (chaos; refined from L0/`exp_chaos_as_engine`): on irreducible micro-dynamics its only lawful purchase is the **`bundle` reading a coarse-grained MACRO statistic** (= empirical characteristic function). Corollary (the architecture): a physics-grounded AI = **[learned encoder DISCOVERS] ⊕ [HDC HOLDS/ROLLS/REVERSES the given structure, deterministically + calibrated + 1-bit]** — never "HDC computes physics."

"Scaling law" is a property of LEARNING, not the algebra (HDC's own scaling = linear capacity, L12); HDC inherits scaling only by stacking on a learned encoder.
