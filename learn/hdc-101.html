Hyperdimensional Computing. For absolute beginners: click any blue word for a plain-language explanation. The vectors below are clickable and computable.超维计算(Hyperdimensional Computing)。给纯小白:每个 蓝词 点一下看人话。下面的向量能点能算。

HDC is another kind of "brain algorithm" that runs on our chip — instead of neural networks, it uses random vectors with tens of thousands of dimensions to "bind / bundle / compare". It is natively 1-bit, noise-tolerant, learns in one shot, and never forgets — which is exactly why it fits ReRAM, hardware that is power-thrifty but not very precise. This page explains what it is, and gathers all of our HDC code repositories in one place.HDC 是我们这块芯片上跑的另一种"大脑算法"——不靠神经网络,靠几万维的随机向量做"绑定/打包/比对"。它天生 1-bit、抗噪、一次就学会、永不遗忘——正好长在 ReRAM 这种又省电又不太精确的硬件上。这页讲清它是什么,并把我们所有 HDC 的代码仓库挂在一起。

① One idea = a string of ten-thousand-dimensional random numbers① 一个想法 = 一串一万维的随机数

HDC represents every concept (say "cat", "red", "king") as a very, very long vector (10K–100K dimensions), where each dimension is just +1 or −1. At the start it is assigned at random.HDC 把每个概念(比如"猫""红色""国王")都表示成一条很长很长(1万~10万维)的向量,每一维只是 +1 或 −1。一开始就是随机给的。

⭐ The magic: any two random long vectors have a similarity near 0 (almost perpendicular). The higher the dimension, the more this holds. It means the space is so vast it can fit almost infinitely many mutually non-interfering concepts — this is the root of everything in HDC. Why?⭐ 神奇之处:任意两个随机长向量,相似度都接近 0(几乎垂直)。维度越高越是如此。这意味着空间大到几乎装得下无穷多个互不干扰的概念——这是 HDC 一切的根。为什么?

② Three operations are enough to build "thinking"② 三个动作,就能搭出"思考"

HDC has only three operations, but they are enough:HDC 只有 三种运算,但够用:

🔗 BIND (snap two things into a pair)🔗 BIND 绑定(把两个东西"扣"成一对)

Multiply A and B element-by-element → you get a new vector C that looks like nothing else. It represents an "A paired with B" key-value pair (e.g. "color = red"). And it is reversible: bind C with B again and you recover A exactly.把 A 和 B 逐位相乘 → 得到一个和谁都不像的新向量 C。用来表示"A 配 B"这种键值对(比如"颜色=红")。而且可逆:拿 C 再和 B 绑一次,就把 A 原样还原出来。

Result C结果 C
Is C like A?C 和 A 像吗

Click "bind": C is unlike both A and B (similarity ≈ 0) — it is something brand new. But no information is lost; unbinding gets it all back.点"绑定"看看:C 和 A、和 B 都不像(相似度≈0)——它是个全新的东西。但信息没丢,解绑能完整拿回来。

📦 BUNDLE (stuff a pile of things into "one memory")📦 BUNDLE 打包(把一堆东西塞进"一个记忆")

Add several vectors and take the sign → you get a vector M that is a little bit like every member. It represents a set / a stretch of memory ("I have seen X, Y, Z"). Later, compare anything against it and you know whether it is in this memory.把几个向量加起来取符号 → 得到一个和每个成员都有点像的向量 M。用来表示集合 / 一段记忆("我见过 X、Y、Z")。之后拿任意东西去比,就知道它在不在这个记忆里。

Memory M记忆 M
Is member X inside?成员 X 在里面?
Is stranger R inside?陌生 R 在里面?

Click "bundle": member X is clearly similar to M (it's in the memory); stranger vector R is ≈ 0 to M (not in it). This is HDC's way of "remembering".点"打包":成员 X 和 M 明显相似(在记忆里),陌生向量 R 和 M ≈0(不在)。这就是 HDC 的"记得"。

↪ PERMUTE (remember "order")↪ PERMUTE 移位(记住"顺序")

Shift the whole vector by a few positions → used to encode sequence / position ("A then B" ≠ "B then A"). This way even sentences and time series fit into a single vector.把向量整体平移几位 → 用来编码先后/位置("先 A 后 B" ≠ "先 B 后 A")。这样连句子、时间序列都能装进一条向量。

"Random" here means a coin flip: each dimension independently gets +1 with 50% chance and −1 with 50% chance. Ten thousand dimensions = ten thousand flips. Precisely because it's independent and random, two vectors generated this way are almost never alike by chance.这里的“随机”就是抛硬币:每一维独立地 50% 给 +1、50% 给 −1。一万维就抛一万次。正因为是独立随机,两个这样生成的向量几乎不可能碰巧很像。
In very high dimensions, for two random ±1 vectors the "same-sign positions" and "opposite-sign positions" are almost exactly half and half, so the dot product ≈ 0 and the similarity ≈ 0 (nearly perpendicular / orthogonal). The higher the dimension, the more stable this "≈ 0" becomes. The payoff: you can pack in a huge number of mutually non-interfering concepts and still tell them apart reliably by "similarity" — that's the superpower of "hyperdimensional".在很高的维度里,两个随机 ±1 向量“同号的位”和“异号的位”几乎各占一半,点积≈0,所以相似度≈0(近乎垂直/正交)。维度越高,这个“≈0”越稳。好处:你可以塞进海量互不干扰的概念,还能用“相似度”可靠地区分它们——这就是“超维”的超能力。
HDC has only three operations: BIND (element-wise multiply, snapping two into a pair, reversible), BUNDLE (sum and take the sign, merging several into one "set/memory" that resembles every member), and PERMUTE (a whole-vector shift, encoding order/position). With just these three moves you can build key-value pairs, sets, and sequences — enough to express structured "ideas".HDC 只有三种运算:BIND 绑定(逐位相乘,把两个扣成一对、且可逆)、BUNDLE 打包(相加取符号,把多个并成一个“集合/记忆”、和每个成员都像)、PERMUTE 移位(整体平移,编码顺序/位置)。就这三招,能搭出键值对、集合、序列——足够表达结构化的“想法”。
