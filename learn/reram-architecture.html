③· Time-domain counter: how do you count a 512-long column accurately? (SPIKA's real trick)③· 时域计数器:512 长列怎么数得准?(SPIKA 真招)

The hard part: a column accumulates hundreds of rows, the current amplitude is large and hard to measure accurately. SPIKA's solution = don't measure amplitude, count clicks: one column for positive weights, one for negative (two-state cells); the current charges a capacitor, and crossing the threshold gives one "click" to the shared counter (+column adds, −column subtracts); the final count = the difference between the two columns. Tap "Read out":难点:一列累加几百行,电流幅值大、难量准。SPIKA 的解法 = 不量幅值,数 click:正权重一列、负权重一列(二态 cell),电流给电容充电,过阈值就"click"一下共享计数器(+列加、−列减),最后计数=两列之差。点"读出":

Tap "Read out": each current-emitting cell in the + column adds +1 to the counter, each in the − column subtracts 1; the final value = column sum (differential).点"读出":+列每个出电流的 cell 让计数器+1,−列每个−1,最后=列和(差分)。

Why is this accurate? (and it answers two questions along the way)为什么这样就准?(顺便答两个问题)

① Count integers, don't measure amplitude数整数,不量幅值Two-state cells: the column sum is essentially "how many +1s minus how many −1s" = an integer. Counting clicks is far more stable than measuring analog current.二态 cell:列和本质是"几个 +1 减几个 −1"=整数。数 click 比量模拟电流稳得多。
② Differential cancels common mode差分消共模Subtracting the ± two columns: temperature drift / IR-drop / aging hit both columns equally → they cancel.±两列相减,温漂/IR-drop/老化对两列一样 → 抵消。
③ Global clock · parallel全局时钟·并行One clock broadcasts to all columns' counters, and all columns count their own clicks simultaneously —— this is "broadcasting time in parallel", and it works.一个时钟广播给所有列的计数器,所有列同时数自己的 click —— 这就是"并行发时间",可以。
④ No atomic clock needed不需要原子钟A GHz on-chip clock is more than enough for µs readout; the accuracy bottleneck is on the analog side (capacitor / threshold deviation), which an atomic clock can't fix and can't fit on chip anyway.GHz 片上时钟对 µs 读出绰绰有余;精度瓶颈在模拟侧(电容/阈值偏差),原子钟修不了它,还上不了片。
⑤ The real trick for large currents大电流的真招 (ISSCC'24 34.8)(ISSCC'24 34.8)Multi-bit input compression (not sent all at once) + HRS bias (most cells are in the low-current 0 state) + residual shared ADC, bounding the dynamic range.输入多bit压缩(不一次发完)+ HRS偏置(多数 cell 是低电流0态)+ 残差共享ADC,把动态范围框住。

③·· RS-ADC: how do multiple columns share one ADC (ISSCC'24 34.8, real chip)③·· RS-ADC:多列怎么共享一个 ADC(ISSCC'24 34.8,真芯片)

Real chips confirm the ADC is shared across columns (not one per column). Residual-shared ADC (RS-ADC): each column does cheap coarse quantization (a comparator), then the residual after coarse quantization is sent via MUX into one shared fine ADC for sequential quantization. ④ macro: multiple sub-arrays + adder-tree merge④ macro:多个子阵列 + 加法树合并

One macro = a few sub-arrays + readout + an adder tree / XOR tree that merges each sub-array's partial sums. Real chip (the binary one): 4 × 512×12 sub-arrays → XOR-tree merge. One macro is roughly ~16–32 Mbit.一个 macro = 几个子阵列 + 读出 + 加法树/XOR 树把各子阵列的部分和合并。真芯片(二元那款):4 个 512×12 子阵列 → XOR 树合并。一个 macro 大概 ~16–32 Mbit。

The first layer of "how 32Mb is assembled": sub-array →(tight on-chip interconnect)→ macro. This layer is transistor-level interconnect, all on one piece (the expert's exact words)."32Mb 怎么拼" 的第一层:子阵列 →(片内紧密互联)→ macro。这一层是晶体管级互联,在一块(专家原话)。

⑤ Tiling multiple cores: area isn't the wall — clock and interconnect are⑤ 多个核拼接:面积不是墙,时钟和互联才是

Beyond ~32Mbit you tile multiple cores (macro/core). Drag the core count to see the cost: more cores mean longer clock-signal routing, harder alignment, more complex scheduling.超过 ~32Mbit 就拼多个核(macro/core)。拖核数看代价:核越多,时钟信号走线越长、越难对齐,调度越复杂。
