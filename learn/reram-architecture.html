ReRAM-CIM 架构探索 🔬 从一个单元到一整片(基于真文献)

"32Mb 怎么拼起来、crossbar 怎么做、ADC 是不是每块都有、怎么黏一起" —— 用真实芯片的架构,一层层点开看。数字都标了文献来源。
📚 真文献锚点(ReRAM = RRAM,神经网络/LLM CIM):
① 清华 28nm 576K RRAM-CIM macro,2T2R 差分,2.82 TOPS/mm²;
② ★ SPIKA(RRAM ML 加速器,64×128 crossbar):三元权重 = 二态单元(LRS=±1 / HRS=0)+ 正负权重放相邻两列 + 每 2 列共享 1 个 5-bit 计数器(时域读出,无 ADC,天然做减法)。180nm,195 TOPS/W;
③ 清华台 ISSCC'24 22nm 16Mb 浮点 ReRAM-CIM(FP16/BF16,28–31 TFLOPS/W);
④ Cascaded ReRAM crossbar for Transformer + ADC 虚拟化(多阵列动态共享 ADC);
⑤ 华为-字节 ISSCC'26 RRAM 芯片(清华,66× CPU)。
⭐ SPIKA 直接指明三元落硅路径:二态单元 + 相邻 ± 列差分 + 共享计数器 → 不用 3 电平 cell,也不用 ADC。

③ ADC 怎么设计?是每列都有吗?——三种方案(核心)

你的问题:"是每块 16×16 都有 ADC 吗?" —— 不是每块,是每"列"读一个电流。但有三种取舍,点切换看面积/密度/速度怎么变:

④ macro:多个子阵列 + 加法树合并

一个 macro = 几个子阵列 + 读出 + 加法树/XOR 树把各子阵列的部分和合并。真芯片(二元那款):4 个 512×12 子阵列 → XOR 树合并。一个 macro 大概 ~16–32 Mbit。

"32Mb 怎么拼" 的第一层:子阵列 →(片内紧密互联)→ macro。这一层是晶体管级互联,在一块(专家原话)。

⑤ 多个核拼接:面积不是墙,时钟和互联才是

超过 ~32Mbit 就拼多个核(macro/core)。拖核数看代价:核越多,时钟信号走线越长、越难对齐,调度越复杂。
