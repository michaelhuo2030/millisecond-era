ReRAM-CIM,从最底层说起 🧱

给完全的外行(包括一个半年前还不写代码的人)。每个 蓝色虚线词 点一下,底部弹出大白话解释。

我们想做一块芯片,把一个 AI 模型的"脑子"直接刻进存储单元里,让它跑得飞快(每秒几千到几万个字)还省电。这块芯片叫 ReRAM-CIM。这页从"什么是一个 bit"开始,一路讲到"它到底能装下多大的模型"——也就是我心里最没底、最想弄清的那个问题。

① 先学会数数:bit、byte、Mb、MB

所有"容量"都是在数同一个东西:能存多少个 bit。

单位等于大白话

byte(字节)8 bit1 个英文字母
Kb / KB1024 bit / 1024 byte一句话
Mb / MB100万 bit / 100万 byte一张图 / 一本书
Gb / GB10亿 bit / 10亿 byte一部电影

🤔 "我们不是用三元组吗,为什么 byte 还是 8 bit?"

好问题,这是两层不同的事——byte=8bit 只是"数容量的尺子",和数据是二元还是三元无关。就像"一打=12 个",不管你数鸡蛋还是苹果都是 12。

层在说什么数值
容量(尺子)芯片能存多少 bit/byte16.32 Mb(固定,byte=8bit)
三元编码我们模型每个权重占几 bit~1.58–2 bit/权重
能装多少权重容量 ÷ 每权重 bit总 bit ÷ 2

三元的好处在第二层:普通权重要 16 bit(fp16),三元只要 ~2 bit → 同容量多塞 8 倍权重。但第一层那把尺子(byte=8bit)本身没变。
为什么 byte 恰好 8 bit?纯历史约定:8 bit=2⁸=256 种,刚好够编一个字符,就定成标准单位了。Mb 和 MB 差 8 倍,根子就在 1 byte=8 bit。

⚠️ 全场最大的坑:小写 b = bit,大写 B = byte,差 8 倍。
举个工程量级:28nm 上单个 ReRAM-CIM macro 大约 16 Mb(megabit)= 只有 2 MB(megabyte)。如果误当成 32 MB,就高估了 16 倍。为什么这要命?

② 什么是一个"存储单元"(cell)?

芯片里存 1 个 bit 的最小零件叫 cell。怎么造它,是 SRAM 和 ReRAM 最根本的区别:

SRAM cell

用 6 个晶体管搭一个"跷跷板",靠通电保持 0/1。
• 快、成熟
• 但占地大(6 个管子存 1 bit)
• 断电就忘(易失)
• 权重每次要重新搬进来

ReRAM cell

用 1 个晶体管 + 1 个电阻开关(叫 1T1R)。
• 小得多(密度高)
• 断电不忘(非易失)
• 权重刻进去就一直在
• 但写得慢、会"老化"、多电平不稳

关键一点:ReRAM 相对 SRAM 只是换了工艺和存储单元,整体架构(尤其外围电路)高度相通——所以做 SRAM-CIM 的经验能很快迁移到 ReRAM-CIM。

③ ReRAM 到底是什么物理?(点一下亲手"造"一个)

ReRAM = Resistive RAM,电阻式存储器。结构简单到不可思议:金属 — 氧化物 — 金属,像个三明治。给它加电压,中间的氧化物里会长出或断开一根导电细丝:

上电极
下电极
高阻 (HRS) = 0

👆 点这个 cell:

长出细丝 → 电阻变低(LRS)→ 电流好通过 → 记作 1(SET)
断开细丝 → 电阻变高(HRS)→ 电流难通过 → 记作 0(RESET)

就这么个开关,反复刻录就能存模型权重。整块芯片是几百万个这样的 cell 排成网格(crossbar 阵列)。

⑤ 多电平(MLC):一个 cell 能不能存更多?

如果导电细丝能调出好几种粗细(几档电阻),一个 cell 就能存不止 1 bit。这叫 MLC。听起来很美——但这正是我最担心、也是 ReRAM 最大的软肋:

🎚 拖一下:一个 cell 想存几档?

能存 2 档 = 1.0 bit/cell 可靠度:高 ✅

这就是三元/HDC 的命门所在。 普通 8-bit DNN 需要一个 cell 稳稳分出 16+ 档——ReRAM 做不到、会飘。但三元只需要 3 档 {−1,0,+1},HDC 二元只需要 2 档。我们故意用低精度算法,去吃 ReRAM 器件不精确的亏——这是整个技术路线的赌注。

🔎 "MLC=3 是不是就是三元组?" 几乎是,但要分清两层:三元 {−1,0,+1} 说的是"值有 3 个"(算法);MLC-3 说的是"一个 cell 物理上分 3 挡"(硬件)。把一个三元权重存进一个 3 挡 cell,二者刚好对上——所以"原生三元 cell"=MLC-3。但三元也可以用 2 个普通 1-bit(SLC)cell 差分来存,不一定非走 MLC。关键是:三元只需要 2–3 挡,正好避开高挡位 MLC 那个最难的区。→ 点这里玩"多电平物理上怎么实现"的交互图,拖到 6–8 挡就能看到挡位挤到一起、开始读错。

⑥ "存算一体"(CIM)是什么黑魔法?

普通电脑:数据存在内存,要算的时候搬到 CPU,算完搬回去——搬运本身又慢又费电(这叫内存墙)。

CIM(Compute-in-Memory,存内计算):干脆在存权重的地方直接做计算,权重一步都不挪。怎么做到?靠两条中学物理:

欧姆定律电流 = 电压 × 电导(=1/电阻)。把输入做成电压、权重做成 cell 电阻 → 电流自动 = 输入×权重。一次乘法,零指令。
基尔霍夫定律一整列 cell 的电流在导线上自动相加 → 一列就是一次"乘加求和"(MAC)。整个矩阵乘法在一瞬间、用物理完成。

所以 CIM 的速度不是看内存带宽(权重不搬),而是看阵列每秒能做多少次这种模拟乘加。代价是:要把模拟电流读成数字,需要大量 ADC——这玩意又大又费电,正是 ReRAM-CIM 里电路设计最难、最吃面积和功耗的地方。

⑦ SRAM-CIM、ReRAM-CIM……各种存储怎么选?

先看最常拿来对比的两种:两者架构、外围、ADC 那套几乎一样,区别只在"用什么 cell 存权重":

维度SRAM-CIMReRAM-CIM(我们的方向)
存储单元6+ 晶体管搭的 SRAM1 晶体管 + 1 电阻开关
断电就忘(易失)→ 权重每次重搬不忘(非易失)→ 权重刻进去常驻 ⭐
密度(同面积存多少)低(管子多)高 3–10×
读/算速度极快、稳快,但模拟读出要 ADC
写入快、随便写慢、会老化(耐久有限)
模拟精度好(数字单元)差(细丝会飘)← 我们用三元绕开
成熟度标准 CMOS,谁都能做要特殊工艺,资料只在论文里
一句话: SRAM-CIM 像"用昂贵的快内存现搬现算",ReRAM-CIM 像"把模型一次性烧进只读卡带,从此权重不动"。对"模型权重很大、又想常驻、又想省电"的 LLM 推理,ReRAM 的非易失 + 高密度是关键优势;它的不精确,正好被三元算法吃下。

再放宽一点:整个存储 / 存内计算家族

世界上不止 SRAM 和 ReRAM。把常见的几种排一排,就明白为什么我们押 ReRAM(也明白别人为什么选别的):

技术断电密度写入适合存内计算?一句话 / 谁在用
SRAM易失低极快·无限次✅ 成熟CPU 缓存;快但占地、断电忘(存算公司主流之一)
DRAM易失中快△ 在研电脑内存条;要不停刷新
NOR Flash非易失中慢✅ 模拟CIM存固件;Mythic 用它做模拟存算
NAND Flash非易失极高(3D 堆叠王)慢·有限次△U盘/SSD;最便宜的大容量
ReRAM(我们)非易失 ⭐高中·有限次✅ 主攻电阻丝;非易失+高密度+容忍低精度
MRAM非易失中快·耐久好△磁存储;耐久好但密度一般
PCM非易失高慢·发热△相变材料;Intel Optane 那一类,IBM 也在做 CIM

看这张表就懂我们的取舍:LLM 权重又大又想常驻又想省电 → 要非易失 + 高密度(排除 SRAM/DRAM)、又要能在原地做模拟乘加(排除纯 NAND)、又能容忍器件不精确(三元/HDC 的低精度正好接得住 ReRAM 的飘)。这几条交集,ReRAM 最顺。但这不是唯一解——NOR Flash(Mythic)、PCM(IBM)都是真在做的路线。

⑧ 那……到底能放下多大的模型?(我心里没底的那个数)

"能放多少"不是一个数,是 6 个因子相乘。把它拆开,就知道该去确认哪几个数:

A
每个 cell 存几 bit — 多电平可靠吗?(第⑤节,我们的三元命门)
B
每 mm² 多少 cell — 28nm 下单元多大
C
阵列有效率 — macro 里真正是 cell 的面积占比(ADC/驱动吃掉 50–80%)← 理论 vs 实操最大落差
D
可用面积 — 良率/reticle 限制下实际能用多大
E
层数 — 能不能 3D 堆叠(目前业界还没人做过 400mm² 的 ReRAM-CIM)

峰值密度(Mb/mm²)= A×B×C。只要拿到 A、C 和"峰值密度"这三个数,前面那个计算器就能填准,心里就有底了。

一个工程量级: 28nm 上单个 ReRAM-CIM macro 大约 16 Mb。要拼到能跑 BERT-base(~110M 参数 ≈ 256 Mb)需要拼 ~16 个这样的 macro——能不能拼、拼到大面积行不行,就是没人验证过的无人区。

millisecond-era · 毫秒纪 · ReRAM-CIM 101 · 2026-06-01
声明:科普向,数字以公开论文为准;材料/论文条目均为公开文献,如有出入以原文为准。
bit(比特)：信息的最小单位，只有 0 或 1，就像一个开关。8 个 bit = 1 个 byte（字节）= 1 个英文字母的存储量。芯片"容量"本质就是在数能存多少个 bit。
小写 b 是 bit，大写 B 是 byte，1 byte = 8 bit。"16.32 Mb" = 16.32 兆比特 = 2.04 兆字节(MB)。如果误读成 32 MB，就把容量高估了约 16 倍——能跑的模型从"几百万参数的小玩具"被错当成"上亿参数的 BERT"。所以拿到任何容量数字，第一句先确认：是 Mb 还是 MB？
三元权重只有 {−1,0,+1} 三个值，理论上用 log₂3 ≈ 1.58 bit 就能表示。但物理上一个 cell 往往要占 2 bit（比如用两个单元做差分、或一个四电平单元只用其中三档）。所以估算容量时按 2 bit/权重比较稳妥，1.58 是最乐观的理论地板。
cell（存储单元）：芯片里存 1 个 bit 的最小零件。SRAM 用 6 个晶体管搭一个，ReRAM 用 1 个晶体管 + 1 个电阻开关。cell 越小，同样面积能存越多，密度越高。
1T1R = 1 Transistor + 1 Resistor，一个晶体管串一个电阻开关。晶体管负责"选中这个单元"，电阻开关负责"存 0 还是 1"。这是 ReRAM 阵列最常见的单元结构，比 SRAM 的 6 管小很多。
导电细丝（conductive filament）：在金属-氧化物-金属三明治里，加电压会让氧化物中的氧原子移动，留下一串"氧空位"连成一根纳米细丝，像在绝缘层里搭了一座桥。桥通（LRS，低阻）= 1，桥断（HRS，高阻）= 0。反复加正/负电压就能反复搭桥/拆桥，于是能反复写入模型权重。它"会飘"——每次搭出来的桥粗细略有不同，这就是 ReRAM 模拟精度差的根源。
LRS = Low Resistance State（低阻态），细丝接通、电流好走，记作 1。反义是 HRS = High Resistance State（高阻态），细丝断开，记作 0。读出时给个小电压看电流大小，就知道存的是 0 还是 1。
crossbar（交叉阵列）：把横线（行）和竖线（列）交叉排成网格，每个交叉点放一个 cell。这种结构让"一整列同时做乘加求和"成为可能——正是存内计算 CIM 的物理基础。几百万个 cell 就这样排成一大片。
MLC = Multi-Level Cell（多电平单元）：让一个 cell 能区分好几档电阻（不只是高/低两档），从而一个单元存不止 1 bit。能省地方，但 ReRAM 的细丝会随时间漂移、不同单元也不一致，档分得越多越容易读错——良率和可靠性是它的死穴。这恰恰是为什么三元/HDC（只需要 2–3 档）特别适配 ReRAM。
深度学习里 8-bit 权重需要一个单元稳稳分出 256 档（或多个单元拼），ReRAM 的细丝漂移让这很难。但三元权重只有 3 个值、HDC 是二元只有 2 个值——只需要 cell 分出 2–3 档，这是 ReRAM 最容易、最可靠做到的。于是"算法故意低精度"和"器件天生不精确"刚好对上。这是 millisecond-era 整个技术押注的核心假设，需要在真硅上验证（见 README）。
内存墙（memory wall）：现代芯片算得很快，但数据要从内存搬到计算单元，这段"搬运"又慢又费电，成了真正的瓶颈。大模型推理时反复搬几个 GB 的权重，大量时间和电力耗在搬运上。CIM 的全部意义就是：权重不搬，在原地算掉，墙就消失了。
MAC = Multiply-Accumulate（乘加）：神经网络最基本的运算——把输入乘以权重再累加。一个 1000×1000 的矩阵乘法就是上百万次 MAC。CIM 用一列 cell 的电流自动相加，一瞬间完成一整列的 MAC，这就是它快的来源。
ADC = Analog-to-Digital Converter（模数转换器）：CIM 在阵列里算出来的是"模拟电流"，要变回数字才能用，靠 ADC 转换。问题是 ADC 又大又费电——在很多 CIM 设计里，ADC 吃掉了大部分面积和功耗（曾测到 ~88% 能耗、~75% 面积）。这正是 CIM 里电路设计最难的部分，也是"理论密度"远高于"实际能放"的主因。
易失 vs 非易失：易失（volatile）= 一断电就忘，SRAM/DRAM 都是，所以每次开机要重新把权重搬进去。非易失（non-volatile）= 断电也记得，ReRAM/闪存是，权重一次刻进去就常驻芯片。对"权重大又想常驻又想省电"的 LLM，非易失是 ReRAM 的关键卖点。
