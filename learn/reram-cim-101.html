ReRAM-CIM, From the Very Bottom Up 🧱ReRAM-CIM,从最底层说起 🧱

For the complete non-expert (including someone who couldn't write code half a year ago). Click any blue dashed word and a plain-language explanation pops up at the bottom.给完全的外行(包括一个半年前还不写代码的人)。每个 蓝色虚线词 点一下,底部弹出大白话解释。

We want to build a chip that engraves an AI model's "brain" directly into the memory cells, so it runs blazingly fast (thousands to tens of thousands of words per second) and sips power. This chip is called ReRAM-CIM. This page starts from "what is a bit" and goes all the way to "exactly how big a model can it hold" — the question I'm most unsure about and most want to nail down.我们想做一块芯片,把一个 AI 模型的"脑子"直接刻进存储单元里,让它跑得飞快(每秒几千到几万个字)还省电。这块芯片叫 ReRAM-CIM。这页从"什么是一个 bit"开始,一路讲到"它到底能装下多大的模型"——也就是我心里最没底、最想弄清的那个问题。

📌 Note one thing first, so you don't trip up: RRAM = ReRAM = resistive memory, they're the same thing. The full English name is Resistive RAM; some write RRAM, some write ReRAM, some call it a memristor — all referring to the same physical device (metal–oxide–metal, storing data via high vs. low resistance). Just like US and USA are the same country. When a paper writes RRAM, just read it as ReRAM.📌 先记一条,免得绊倒:RRAM = ReRAM = 阻变存储器,是同一个东西。 英文全称都是 Resistive RAM,有人写 RRAM、有人写 ReRAM、有人叫 memristor(忆阻器)——指同一个物理器件(金属–氧化物–金属,靠电阻高低存数据)。就像 US 和 USA 是同一个国家。看到论文写 RRAM,直接当 ReRAM 读。

The benefit of ternary is in the second layer: an ordinary weight needs 16 bit (fp16), but ternary needs only ~2 bit → you can cram in 8× more weights at the same capacity. But the ruler in the first layer (byte=8bit) itself doesn't change.
Why is a byte exactly 8 bit? Pure historical convention: 8 bit = 2⁸ = 256 possibilities, just enough to encode one character, so it became the standard unit. Mb and MB differ by 8×, and the root of that is 1 byte = 8 bit.三元的好处在第二层:普通权重要 16 bit(fp16),三元只要 ~2 bit → 同容量多塞 8 倍权重。但第一层那把尺子(byte=8bit)本身没变。
为什么 byte 恰好 8 bit?纯历史约定:8 bit=2⁸=256 种,刚好够编一个字符,就定成标准单位了。Mb 和 MB 差 8 倍,根子就在 1 byte=8 bit。

② What is a "memory cell"?② 什么是一个"存储单元"(cell)?

The smallest part in a chip that stores 1 bit is called a cell. How you build it is the most fundamental difference between SRAM and ReRAM:芯片里存 1 个 bit 的最小零件叫 cell。怎么造它,是 SRAM 和 ReRAM 最根本的区别:

SRAM cell

Uses 6 transistors to build a "seesaw," holding 0/1 as long as it's powered.
• fast, mature
• but takes up a lot of space (6 transistors to store 1 bit)
• forgets when power is off (volatile)
• weights must be reloaded every time用 6 个晶体管搭一个"跷跷板",靠通电保持 0/1。
• 快、成熟
• 但占地大(6 个管子存 1 bit)
• 断电就忘(易失)
• 权重每次要重新搬进来

ReRAM cell

Uses 1 transistor + 1 resistive switch (called 1T1R).
• much smaller (high density)
• remembers when power is off (non-volatile)
• once weights are engraved, they stay
• but slow to write, "ages," and multi-level is unstable用 1 个晶体管 + 1 个电阻开关(叫 1T1R)。
• 小得多(密度高)
• 断电不忘(非易失)
• 权重刻进去就一直在
• 但写得慢、会"老化"、多电平不稳

Key point: relative to SRAM, ReRAM only swaps out the process and the memory cell — the overall architecture (especially the peripheral circuits) is highly shared. So experience building SRAM-CIM transfers quickly to ReRAM-CIM.关键一点:ReRAM 相对 SRAM 只是换了工艺和存储单元,整体架构(尤其外围电路)高度相通——所以做 SRAM-CIM 的经验能很快迁移到 ReRAM-CIM。

③ What physics is ReRAM, exactly? (click to "build" one yourself)③ ReRAM 到底是什么物理?(点一下亲手"造"一个)

ReRAM = Resistive RAM, a resistance-based memory. Its structure is unbelievably simple: metal — oxide — metal, like a sandwich. Apply a voltage, and inside the middle oxide a conductive filament grows or breaks:ReRAM = Resistive RAM,电阻式存储器。结构简单到不可思议:金属 — 氧化物 — 金属,像个三明治。给它加电压,中间的氧化物里会长出或断开一根导电细丝:

Top electrode上电极
Bottom electrode下电极
High resistance (HRS) = 0高阻 (HRS) = 0

👆 Click this cell:👆 点这个 cell:

Grow a filament → resistance drops (LRS) → current flows easily → recorded as 1 (SET)
Break the filament → resistance rises (HRS) → current flows poorly → recorded as 0 (RESET)长出细丝 → 电阻变低(LRS)→ 电流好通过 → 记作 1(SET)
断开细丝 → 电阻变高(HRS)→ 电流难通过 → 记作 0(RESET)

That's the whole switch — engrave it over and over and you can store model weights. The whole chip is millions of such cells arranged in a grid (a crossbar array).就这么个开关,反复刻录就能存模型权重。整块芯片是几百万个这样的 cell 排成网格(crossbar 阵列)。

⑤ Multi-level (MLC): can one cell store more?⑤ 多电平(MLC):一个 cell 能不能存更多?

If the conductive filament can be tuned to several different thicknesses (several resistance levels), one cell can store more than 1 bit. This is called MLC. It sounds great — but this is exactly what I worry about most, and ReRAM's biggest weakness:如果导电细丝能调出好几种粗细(几档电阻),一个 cell 就能存不止 1 bit。这叫 MLC。听起来很美——但这正是我最担心、也是 ReRAM 最大的软肋:

🎚 Drag it: how many levels should one cell store?🎚 拖一下:一个 cell 想存几档?

Stores 能存 2 levels = 档 = 1.0 bit/cell Reliability: high ✅

This is the crux for ternary/HDC. An ordinary 8-bit DNN needs one cell to reliably resolve 16+ levels — ReRAM can't, it drifts. But ternary has only 3 values {−1,0,+1}, and binary HDC has only 2 values. Key: we do not rely on "tuning many levels in one cell" (that drifts) — instead we use two binary cells in a differential pair + a digital ADC-free counter readout: avoiding both high-level MLC and analog ADCs. Low-precision algorithm + binary cells + digital readout — this is the bet of the entire technical roadmap.这就是三元/HDC 的命门所在。 普通 8-bit DNN 需要一个 cell 稳稳分出 16+ 档——ReRAM 做不到、会飘。但三元只有 3 个值 {−1,0,+1},HDC 二元只有 2 个值。关键:我们不靠"一个 cell 调多档"(那会飘)——而是用两个二态 cell 差分 + 数字无-ADC 计数器读出:既避开高档位 MLC、又避开模拟 ADC。低精度算法 + 二态 cell + 数字读出,这是整个技术路线的赌注。

🔎 "Is MLC=3 the same as ternary?" In theory it lines up (one cell split into 3 levels storing one ternary value), but we don't go this way: many levels in one cell drift and are hard to read. We take a different route: store ternary using 2 ordinary binary (SLC) cells in a differential pair (one for the + column, one for the − column, read their difference). Binary cells are the easiest to read and most stable, and can be read out with a digital counter, ADC-free. So this project's "ternary in silicon" = binary differential, not MLC-3. → Click here for the interactive diagram of "two binary cells in a differential pair" (the MLC diagram in the next level just demonstrates "why many levels drift, and why we don't go there").🔎 "MLC=3 是不是就是三元?" 理论上能对上(单 cell 分 3 档存一个三元值),但我们不走这条:单 cell 多档会飘、难读。我们走的是另一条:用 2 个普通二态(SLC)cell 差分存三元(+列一个、−列一个,读它俩的差),二态 cell 最好读、最稳,还能用数字计数器无-ADC 读出。所以本项目的"三元落硅"= 二态差分,不是 MLC-3。→ 点这里看"两个二态 cell 差分"的交互图(下一档的 MLC 图只是演示"为什么多档会飘、我们为什么不走")。

⑥ What black magic is "compute-in-memory" (CIM)?⑥ "存算一体"(CIM)是什么黑魔法?

An ordinary computer: data lives in memory, and when you need to compute you move it to the CPU, then move the result back — the moving itself is slow and power-hungry (this is the memory wall).普通电脑:数据存在内存,要算的时候搬到 CPU,算完搬回去——搬运本身又慢又费电(这叫内存墙)。

CIM (Compute-in-Memory): simply do the computation right where the weights are stored, without moving the weights a single step. How? With two pieces of high-school physics:CIM(Compute-in-Memory,存内计算):干脆在存权重的地方直接做计算,权重一步都不挪。怎么做到?靠两条中学物理:

Ohm's law欧姆定律current = voltage × conductance (=1/resistance). Make the input a voltage and the weight a cell resistance → current automatically = input × weight. One multiplication, zero instructions.电流 = 电压 × 电导(=1/电阻)。把输入做成电压、权重做成 cell 电阻 → 电流自动 = 输入×权重。一次乘法,零指令。
Kirchhoff's law基尔霍夫定律the currents of a whole column of cells add up automatically on the wire → one column is one "multiply-accumulate sum" (MAC). The entire matrix multiplication is done in an instant, by physics.一整列 cell 的电流在导线上自动相加 → 一列就是一次"乘加求和"(MAC)。整个矩阵乘法在一瞬间、用物理完成。

So CIM's speed is not about memory bandwidth (weights don't move), but about how many such multiply-accumulates the array can do per second. Analog CIM has to read the current back into digital and needs lots of ADCs (big and power-hungry, often eating up most of the area/power) — this is exactly why we go with "binary differential + digital counter" readout, ADC-free, sparing ourselves this hardest part.所以 CIM 的速度不是看内存带宽(权重不搬),而是看阵列每秒能做多少次这种乘加。模拟 CIM 要把电流读成数字、需要大量 ADC(又大又费电,常吃掉大半面积/功耗)——我们正是为此走"二态差分 + 数字计数器"读出、无 ADC,把这块最难的省掉。

bit: the smallest unit of information, just 0 or 1, like a switch. 8 bits = 1 byte = the storage of one English letter. A chip's "capacity" is essentially counting how many bits it can store.bit(比特)：信息的最小单位，只有 0 或 1，就像一个开关。8 个 bit = 1 个 byte（字节）= 1 个英文字母的存储量。芯片"容量"本质就是在数能存多少个 bit。
Lowercase b is bit, uppercase B is byte, 1 byte = 8 bit. "16.32 Mb" = 16.32 megabit = 2.04 megabyte (MB). If you misread it as 32 MB, you've overestimated capacity by about 16× — and the model you can run gets mistaken from a "small few-million-parameter toy" for a "hundred-million-parameter BERT." So with any capacity number, your first question is: is it Mb or MB?小写 b 是 bit，大写 B 是 byte，1 byte = 8 bit。"16.32 Mb" = 16.32 兆比特 = 2.04 兆字节(MB)。如果误读成 32 MB，就把容量高估了约 16 倍——能跑的模型从"几百万参数的小玩具"被错当成"上亿参数的 BERT"。所以拿到任何容量数字，第一句先确认：是 Mb 还是 MB？
A ternary weight has only 3 values {−1,0,+1}, which in theory can be represented with log₂3 ≈ 1.58 bit. But physically a cell often takes 2 bit (e.g. using two cells for a differential pair, or using only 3 of a 4-level cell's levels). So when estimating capacity it's safer to use 2 bit/weight; 1.58 is the most optimistic theoretical floor.三元权重只有 {−1,0,+1} 三个值，理论上用 log₂3 ≈ 1.58 bit 就能表示。但物理上一个 cell 往往要占 2 bit（比如用两个单元做差分、或一个四电平单元只用其中三档）。所以估算容量时按 2 bit/权重比较稳妥，1.58 是最乐观的理论地板。
cell (memory cell): the smallest part in a chip that stores 1 bit. SRAM builds one from 6 transistors; ReRAM uses 1 transistor + 1 resistive switch. The smaller the cell, the more you can store in the same area, the higher the density.cell（存储单元）：芯片里存 1 个 bit 的最小零件。SRAM 用 6 个晶体管搭一个，ReRAM 用 1 个晶体管 + 1 个电阻开关。cell 越小，同样面积能存越多，密度越高。
1T1R = 1 Transistor + 1 Resistor, one transistor in series with one resistive switch. The transistor "selects this cell," the resistive switch "stores 0 or 1." This is the most common cell structure in ReRAM arrays, much smaller than SRAM's 6 transistors.1T1R = 1 Transistor + 1 Resistor，一个晶体管串一个电阻开关。晶体管负责"选中这个单元"，电阻开关负责"存 0 还是 1"。这是 ReRAM 阵列最常见的单元结构，比 SRAM 的 6 管小很多。
conductive filament: inside the metal-oxide-metal sandwich, applying a voltage makes oxygen atoms in the oxide move, leaving a chain of "oxygen vacancies" that connect into a nanoscale filament, like building a bridge across the insulating layer. Bridge connected (LRS, low resistance) = 1, bridge broken (HRS, high resistance) = 0. Repeatedly applying positive/negative voltage builds/breaks the bridge over and over, so you can repeatedly write model weights. It "drifts" — each bridge comes out slightly different in thickness, which is the root cause of ReRAM's poor analog precision.导电细丝（conductive filament）：在金属-氧化物-金属三明治里，加电压会让氧化物中的氧原子移动，留下一串"氧空位"连成一根纳米细丝，像在绝缘层里搭了一座桥。桥通（LRS，低阻）= 1，桥断（HRS，高阻）= 0。反复加正/负电压就能反复搭桥/拆桥，于是能反复写入模型权重。它"会飘"——每次搭出来的桥粗细略有不同，这就是 ReRAM 模拟精度差的根源。
LRS = Low Resistance State, the filament is connected, current flows easily, recorded as 1. Its opposite is HRS = High Resistance State, the filament is broken, recorded as 0. To read, apply a small voltage and look at the current magnitude to know whether 0 or 1 is stored.LRS = Low Resistance State（低阻态），细丝接通、电流好走，记作 1。反义是 HRS = High Resistance State（高阻态），细丝断开，记作 0。读出时给个小电压看电流大小，就知道存的是 0 还是 1。
crossbar (crossbar array): horizontal lines (rows) and vertical lines (columns) crossed into a grid, with a cell at each crossing point. This structure makes "a whole column doing multiply-accumulate at once" possible — exactly the physical basis of compute-in-memory (CIM). Millions of cells are arranged into one big sheet this way.crossbar（交叉阵列）：把横线（行）和竖线（列）交叉排成网格，每个交叉点放一个 cell。这种结构让"一整列同时做乘加求和"成为可能——正是存内计算 CIM 的物理基础。几百万个 cell 就这样排成一大片。
MLC = Multi-Level Cell: lets one cell distinguish several resistance levels (not just high/low two levels), so one cell stores more than 1 bit. It saves space, but ReRAM's filament drifts over time and varies cell to cell — the more levels you split into, the more likely a read error — so yield and reliability are its Achilles' heel. This is exactly why ternary/HDC (needing only 2–3 levels) fits ReRAM especially well.MLC = Multi-Level Cell（多电平单元）：让一个 cell 能区分好几档电阻（不只是高/低两档），从而一个单元存不止 1 bit。能省地方，但 ReRAM 的细丝会随时间漂移、不同单元也不一致，档分得越多越容易读错——良率和可靠性是它的死穴。这恰恰是为什么三元/HDC（只需要 2–3 档）特别适配 ReRAM。
In deep learning, an 8-bit weight needs one cell to reliably resolve 256 levels (or multiple cells combined), which ReRAM's filament drift makes very hard. But ternary has only 3 values and binary HDC only 2 — we do not use a single multi-level cell (it drifts), but instead use two binary cells in a differential pair to store one ternary value (each cell only needs to reliably distinguish "on/off" two states, which is what ReRAM does most easily), then read it out with a digital counter, ADC-free. So "low-precision algorithm + binary cells + digital readout" lines up, avoiding both high-level MLC and analog ADCs. This is millisecond-era's core bet, to be verified on real silicon (see README).深度学习里 8-bit 权重需要一个单元稳稳分出 256 档（或多个单元拼），ReRAM 的细丝漂移让这很难。但三元只有 3 个值、HDC 二元只有 2 个值——我们不用单个多档 cell（会飘），而是用两个二态 cell 差分存一个三元值（每个 cell 只需可靠区分"开/关"两态，这是 ReRAM 最容易做到的），再用数字计数器无-ADC 读出。于是"算法低精度 + 二态 cell + 数字读出"对上，既避开高档位 MLC、又避开模拟 ADC。这是 millisecond-era 的核心押注，需在真硅上验证（见 README）。
memory wall: modern chips compute very fast, but data must be moved from memory to the compute units, and this "moving" is slow and power-hungry, becoming the real bottleneck. During large-model inference, several GB of weights are moved back and forth repeatedly, and a huge amount of time and power is spent on moving. The whole point of CIM is: don't move the weights, compute in place, and the wall disappears.内存墙（memory wall）：现代芯片算得很快，但数据要从内存搬到计算单元，这段"搬运"又慢又费电，成了真正的瓶颈。大模型推理时反复搬几个 GB 的权重，大量时间和电力耗在搬运上。CIM 的全部意义就是：权重不搬，在原地算掉，墙就消失了。
MAC = Multiply-Accumulate: the most basic operation in neural networks — multiply the input by the weight and accumulate. A 1000×1000 matrix multiplication is millions of MACs. CIM adds up the currents of a column of cells automatically, completing a whole column's MAC in an instant — that's the source of its speed.MAC = Multiply-Accumulate（乘加）：神经网络最基本的运算——把输入乘以权重再累加。一个 1000×1000 的矩阵乘法就是上百万次 MAC。CIM 用一列 cell 的电流自动相加，一瞬间完成一整列的 MAC，这就是它快的来源。
ADC = Analog-to-Digital Converter: what CIM computes inside the array is an "analog current," which must be turned back into digital to be usable, via an ADC. The problem is ADCs are big and power-hungry — in many CIM designs, ADCs eat up most of the area and power (once measured at ~88% energy, ~75% area). This is the hardest part of circuit design in CIM, and the main reason "theoretical density" is far higher than "what can actually fit."ADC = Analog-to-Digital Converter（模数转换器）：CIM 在阵列里算出来的是"模拟电流"，要变回数字才能用，靠 ADC 转换。问题是 ADC 又大又费电——在很多 CIM 设计里，ADC 吃掉了大部分面积和功耗（曾测到 ~88% 能耗、~75% 面积）。这正是 CIM 里电路设计最难的部分，也是"理论密度"远高于"实际能放"的主因。
volatile vs non-volatile: volatile = forgets the moment power is off, like SRAM/DRAM, so weights must be reloaded at every boot. non-volatile = remembers even with power off, like ReRAM/flash, where weights are engraved once and stay resident on the chip. For LLMs where "weights are big, you want them resident, and you want to save power," non-volatility is ReRAM's key selling point.易失 vs 非易失：易失（volatile）= 一断电就忘，SRAM/DRAM 都是，所以每次开机要重新把权重搬进去。非易失（non-volatile）= 断电也记得，ReRAM/闪存是，权重一次刻进去就常驻芯片。对"权重大又想常驻又想省电"的 LLM，非易失是 ReRAM 的关键卖点。
