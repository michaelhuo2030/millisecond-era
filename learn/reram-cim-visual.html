⓪ CIM vs a traditional computer — what's the difference? (see the big picture first)⓪ CIM 和传统电脑,差在哪?(先看这个大局)

This is the "why" behind the whole thing. Traditional computer: weights live in memory, and to compute you must haul them trip after trip to the compute unit, then haul the results back — moving data is slow and power-hungry (this is the memory wall). CIM: the weights don't move; they're computed in place. Click to watch both run side by side:这是整件事的"为什么"。传统电脑:权重存在内存里,要算就得把它们一趟趟搬到计算单元,算完再搬回——搬运又慢又费电(这叫内存墙)。CIM:权重不动,在原地算掉。 点一下,看两边同时跑:

① What does a single ternary weight look like?① 一个三元权重长什么样?

An ordinary model's weight is a fractional number (needs 16 bits to store). Our ternary weight has only three values: −1 / 0 / +1. Key point: we don't rely on "tuning one cell to three levels" (multi-level cells are hard to read and tend to drift — see the next section) — instead we use two ordinary "on/off" binary cells: one for the "+ column", one for the "− column", and use their difference to set −1 / 0 / +1 (the SPIKA differential-column scheme, = the path real chips take). Click below to toggle:普通模型的权重是个小数(要 16 位存)。我们的三元权重只有三种值:−1 / 0 / +1。关键:我们不靠"一个 cell 调三档"(多电平 cell 难读又会飘,见下一节)——而是用两个普通的"开/关"二态 cell:一个放"+列"、一个放"−列",用它俩的差摆出 −1 / 0 / +1(SPIKA 差分列方案,= 真芯片走的路)。点下面切换看:

①· Why do we not let one cell store multiple levels (MLC)?①· 为什么我们不让一个 cell 存多档(MLC)?

Another path is to make one cell store 2/3/4…8 different values (multi-level, MLC): by writing the conductive filament to different "thicknesses" to get different resistance levels. But the more levels and the narrower the gaps, the more a single ReRAM drift causes a misread — which is exactly why we avoid multi-level and instead use the previous section's "two binary cells, differential" (binary cells are the easiest to read and the most stable). Drag the slider to see how the levels get squeezed, and why we won't even gamble on 3 levels:另一条路是让一个 cell 存 2/3/4…8 个不同的值(多电平 MLC):靠把导电丝写成不同"粗细"得到不同电阻挡位。但挡位越多、间隔越窄,ReRAM 一飘就读错——这正是我们不走多电平、而用上一节"两个二态 cell 差分"的原因(二态 cell 最好读、最稳)。拖滑块看挡位怎么被挤窄、为什么连 3 档我们都不赌:

③ When one computation runs, what happens inside? (the core!)③ 做一次计算时,里面发生了什么?(核心!)

This is CIM's magic. Feed inputs by row on the left (lit = input 1), press "▶ Run one computation": voltage flows in along each row, each cell turns "input × weight" into current, and currents in the same column add up automatically on the bitline (Kirchhoff's law) → the bottom is that column's output. A whole matrix multiply, done in an instant by physics.这是 CIM 的魔法。左边按行给输入(点亮 = 输入 1),按"▶ 跑一次计算":电压沿每行流进去,每个 cell 把"输入×权重"变成电流,同一列的电流在竖线上自动相加(基尔霍夫定律)→ 底下就是这一列的输出。一整个矩阵乘法,用物理一瞬间算完。

input voltage (rows)输入电压(行)+ current+电流− current−电流bottom = column-sum output y底部 = 列求和输出 y
Ordinary chip: weights must be hauled to the compute unit one by one and multiply-accumulated one at a time. CIM: weights don't move, a whole column computes at once — that's the root of its speed and power savings.普通芯片:要把权重一个个搬到计算单元,逐个乘加。CIM:权重不动,一列同时算完 —— 这就是它快和省电的根。

④· Readout method: why can "low precision" remove the ADC?④· 读出方式:为什么"低精度"能把 ADC 拆掉?

To be clear: the array physics does the math (currents add up automatically); the ADC is just a "meter" — it measures that analog current into a digital number. Drag the slider: the higher the readout precision, the bigger and more power-hungry the meter, and the more useful area it eats; down at 1-bit, it shrinks to a tiny "sense amplifier".澄清:算账的是阵列物理(电流自动相加),ADC 只是"读数器"——把那条模拟电流量成数字。拖滑块看:要读的精度越高,读数器越大越费电,有用面积被它吃掉;低到 1-bit,它缩成一个小小的"灵敏放大器"。

There's a third path too: all-digital CIM. Just skip analog readout — put the multiply-accumulate directly into digital circuits (all-digital CIM, like Yizhu's ReRAM scheme, or TSMC's digital SRAM-CIM). You give up a bit of analog's peak energy efficiency in exchange for no ADC at all + more robustness. For low precision like ternary/binary, which is more worthwhile — "analog + 1-bit sense amp" vs "all-digital" — is a real question, and a great one to ask an expert.还有第三条路:全数字 CIM。 干脆不要模拟读出——把乘加直接放数字电路做(全数字 CIM,如亿铸的 ReRAM 方案、台积电的数字 SRAM-CIM)。牺牲一点模拟极致能效,换彻底没有 ADC + 更鲁棒。对三元/二值这种低精度,"模拟+1bit灵敏放大器" vs "全数字" 谁更划算,是个真问题,正好问行家。

An LLM is a stack of transformer layers, each layer being a few big weight matrices. One token passes through each layer from bottom to top; passing one layer = doing one big matrix multiply on that layer's weight matrix (exactly the kind of computation in ③). ADC = Analog-to-Digital Converter: what CIM computes inside the array is a continuous "analog current" (larger or smaller), but a computer only understands digits 0/1. The ADC is the small circuit that "measures the analog current and reports a number". The trouble: it's big and power-hungry — in a CIM chip, the ADCs often eat up more than half the area and power. The more current levels you need to measure (multi-level), the more precise and larger the ADC must be. So it is the number-one reason that "theoretical density is high but you can't actually fit that much".ADC = 模数转换器（Analog-to-Digital Converter）：CIM 在阵列里算出来的是连续的“模拟电流”（有大有小），但电脑只认数字 0/1。ADC 就是那个“把模拟电流量一量、报成一个数字”的小电路。麻烦在于：它又大又费电——一个 CIM 芯片里，ADC 常常吃掉一大半的面积和功耗。要测的电流挡位越多（多电平），ADC 就要越精密越大。所以它是“理论密度高、实际放不下那么多”的头号原因。
macro (compute-in-memory block): a "small building block that does work on its own" — inside is a cell array + supporting ADC/drivers/decoder. One macro stores roughly 16 Mb of weights. A real chip tiles many macros together.macro（存算宏）：一块“能自己干活的小积木”——里面是一片 cell 阵列 + 配套的 ADC/驱动/译码。一个 macro 大约能存 16 Mb 权重。真芯片是把很多个 macro 拼起来用。
cell (storage unit): the smallest part that stores 1 weight. A ReRAM cell = 1 transistor + 1 resistive switch, storing −1/0/+1 by tuning the thickness of the resistive filament.cell（存储单元）：存 1 个权重的最小零件。ReRAM 的 cell = 1 个晶体管 + 1 个电阻开关，靠调电阻丝粗细存 −1/0/+1。
wordline / bitline (rows / columns): the metal wires in a crossbar array. The horizontal line (wordline) feeds input voltage into each row from the left; the vertical line (bitline) collects and adds up a whole column's current at the bottom. A cell sits at each crossing.wordline / bitline（横线 / 竖线）：crossbar 阵列里的金属导线。横线（wordline）从左边把输入电压送进每一行；竖线（bitline）在底部把一整列的电流收集起来相加。交叉点放一个 cell。
MAC = Multiply-Accumulate: the most basic action in a neural network — multiply inputs by weights, then add up a pile of products. One matrix multiply is millions of MACs. CIM adds a column of currents automatically, finishing a whole column's MACs in an instant.MAC = 乘加（Multiply-Accumulate）：神经网络最基本的动作——把输入乘以权重，再把一堆乘积加起来。一次矩阵乘法就是几百万次 MAC。CIM 用一列电流自动相加，一瞬间做完一整列的 MAC。
Kirchhoff's current law: a line of middle-school physics — currents flowing into the same wire add up automatically. CIM borrows this law to let a whole column of cells' currents add themselves up on the bitline, with no instruction, completing the "sum".基尔霍夫电流定律：一句中学物理——汇到同一根导线上的电流会自动相加。CIM 借这条定律，让一整列 cell 的电流在竖线上自己加起来，不用任何指令，就完成了“求和”。
die (bare chip): a small piece cut from a whole wafer — the "body of one chip". Area is usually measured in mm² (square millimeters), e.g. 280/380/450 mm².die（裸片）：从一整片晶圆上切下来的一小块芯片，就是“一颗芯片的本体”。面积常用 mm²（平方毫米）算，比如 280/380/450 mm²。
chiplet: splitting one big chip into several smaller pieces to manufacture, each piece called a chiplet. Small pieces have higher yield and are cheaper, then are tiled together for use. AMD/Apple's big chips all do this.chiplet（芯粒）：把一颗大芯片拆成几块小的来造，每块叫一个 chiplet。小块良率高、更便宜，再拼到一起用。AMD/苹果的大芯片都这么干。
interposer: the "base board" in 2.5D packaging — several chiplets sit side by side on it, and it connects them with extremely dense wiring. Like gluing several blocks onto the same base.interposer（转接板）：2.5D 封装里那块“底板”——几个 chiplet 并排坐在它上面，它负责把彼此用极密的线连通。像把几块积木粘到同一块底座上。
TSV = Through-Silicon Via: a tiny vertical hole drilled through the chip, letting several stacked chip layers connect directly. 3D stacking relies on it to pass signals up and down.TSV = 硅通孔（Through-Silicon Via）：在芯片上垂直打的微小通孔，让上下叠起来的几层芯片能直接连通。3D 堆叠就靠它上下传信号。
2.5D vs 3D: both are "tile many chips" packaging. 2.5D = several chips side by side on an interposer (laid flat); 3D = several chip layers stacked vertically, connected up and down by TSVs. 3D has the highest density for the same area, but heat is hard to dissipate and yield is harder, so "large-area 3D" is still rarely done.2.5D vs 3D：都是“拼很多芯片”的封装。2.5D=几块芯片并排坐在转接板上（平铺）；3D=几层芯片垂直叠起来用 TSV 上下连。3D 同样面积密度最高，但热量难散、良率更难，所以“大面积 3D”至今很少有人做。
yield: the fraction of manufactured chips that are "good and usable". Wafers have random defects, and the bigger the chip area and the more macros per chip, the lower the probability the whole chip is good, so yield drops → cost rises. This is the natural price of "big chips".yield（良率）：造出来的芯片里“合格能用”的比例。晶圆上有随机缺陷，芯片面积越大、单片里 macro 越多，整片全好的概率越低，良率越低 → 成本越高。这是“大芯片”的天然代价。
redundancy repair: build some spare rows/columns/macros, then use the good ones to replace the bad ones after manufacturing. It rescues some chips that would otherwise be scrapped, raising yield.冗余修复（redundancy）：多造一些备用的行/列/macro，造好后用好的去顶替坏的。能把一部分本来报废的芯片救回来，提升良率。
transformer layer: the basic building block of a modern LLM, a stack of identically structured layers. Each layer is mainly a few big weight matrices (attention + feed-forward). A token passes through each layer from bottom to top, doing one big matrix multiply per layer (= a pile of MACs). The more layers and the bigger the matrices, the stronger — and bigger — the model.transformer 层：现代 LLM 的基本积木，一摞同样结构的层叠起来。每层主要就是几个大权重矩阵（注意力 + 前馈）。一个字从下往上穿过每一层，每层做一次大矩阵乘（=一堆 MAC）。层数越多、矩阵越大，模型越强也越大。
all-digital CIM: another route that avoids the ADC — instead of computing in the "analog current domain", put the multiply-accumulate into digital circuits (digital SRAM-CIM, all-digital ReRAM-CIM). Upside: no analog ADC needed at all; results are reproducible, more robust, easier to port. Cost: you lose the extreme per-watt energy efficiency of analog compute-in-memory. For low precision like binary/ternary, all-digital is often very worthwhile; examples include Yizhu (all-digital ReRAM-CIM) and TSMC's digital SRAM-CIM.全数字 CIM：另一条避开 ADC 的路线——不在“模拟电流域”算，而是把乘加放到数字电路里做（数字 SRAM-CIM、全数字 ReRAM-CIM）。好处：完全不需要模拟 ADC，结果可复现、更鲁棒、易移植。代价：失去模拟存内计算那种极致的每瓦能效。对二值/三值这种低精度，全数字往往很划算；代表有亿铸（全数字 ReRAM-CIM）、台积电的数字 SRAM-CIM。
memory wall: modern chips compute blazingly fast, but data must be hauled from memory to the compute unit, and that "hauling" is slow and power-hungry — it has become the real bottleneck. During large-model inference, several GB of weights are hauled back and forth repeatedly, and most of the time and power is spent on moving, not computing. The whole point of CIM is: don't move the weights, compute in place, and the wall disappears.内存墙（memory wall）：现代芯片算得飞快，但数据要从内存搬到计算单元，这段“搬运”又慢又费电，成了真正的瓶颈。大模型推理时要反复搬几个 GB 的权重，大量时间和电力耗在搬运上，而不是计算上。CIM 的全部意义就是：权重不搬，在原地算掉，墙就消失了。
