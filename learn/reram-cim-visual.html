① 一个三元权重长什么样?

①· 多电平(MLC)物理上是怎么实现的?

怎么让一个 cell 存出 2/3/4…8 个不同的值?物理上就一招:把导电丝长成不同的"粗细"。丝越粗 → 电阻越小 → 电流越大。靠控制写入时的电流/电压,把丝停在不同粗细,就得到不同的电阻挡位(电平)。拖滑块选"想要几档",看挡位怎么被挤窄:

② 一群 cell 排成 crossbar 阵列

把横线(行/wordline)和竖线(列/bitline)交叉成网格,每个交叉点放一个三元 cell。这就是 crossbar。点任意 cell 循环 −1/0/+1;换个尺寸看它怎么变大:

+1 导通0 断开−1 反向
横线=输入进来的方向,竖线=电流汇总出去的方向。一个 16×16 就是 256 个权重。真芯片一个 macro 是几百万个。

③ 做一次计算时,里面发生了什么?(核心!)

这是 CIM 的魔法。左边按行给输入(点亮 = 输入 1),按"▶ 跑一次计算":电压沿每行流进去,每个 cell 把"输入×权重"变成电流,同一列的电流在竖线上自动相加(基尔霍夫定律)→ 底下就是这一列的输出。一整个矩阵乘法,用物理一瞬间算完。

输入电压(行)+电流−电流底部 = 列求和输出 y
普通芯片:要把权重一个个搬到计算单元,逐个乘加。CIM:权重不动,一列同时算完 —— 这就是它快和省电的根。

⑦ 一个 LLM 长什么样?在芯片里怎么存怎么算?

LLM 是一摞 transformer 层叠起来,每层就是几个大权重矩阵。一个字(token)从下往上穿过每一层,每穿一层 = 在那层的权重矩阵上做一次大矩阵乘(就是 ③ 那种计算)。拖层数/尺寸,看它需要多少 macro 来存:

ADC = 模数转换器（Analog-to-Digital Converter）：CIM 在阵列里算出来的是连续的“模拟电流”（有大有小），但电脑只认数字 0/1。ADC 就是那个“把模拟电流量一量、报成一个数字”的小电路。麻烦在于：它又大又费电——一个 CIM 芯片里，ADC 常常吃掉一大半的面积和功耗。要测的电流挡位越多（多电平），ADC 就要越精密越大。所以它是“理论密度高、实际放不下那么多”的头号原因。
macro（存算宏）：一块“能自己干活的小积木”——里面是一片 cell 阵列 + 配套的 ADC/驱动/译码。一个 macro 大约能存 16 Mb 权重。真芯片是把很多个 macro 拼起来用。
cell（存储单元）：存 1 个权重的最小零件。ReRAM 的 cell = 1 个晶体管 + 1 个电阻开关，靠调电阻丝粗细存 −1/0/+1。
wordline / bitline（横线 / 竖线）：crossbar 阵列里的金属导线。横线（wordline）从左边把输入电压送进每一行；竖线（bitline）在底部把一整列的电流收集起来相加。交叉点放一个 cell。
MAC = 乘加（Multiply-Accumulate）：神经网络最基本的动作——把输入乘以权重，再把一堆乘积加起来。一次矩阵乘法就是几百万次 MAC。CIM 用一列电流自动相加，一瞬间做完一整列的 MAC。
基尔霍夫电流定律：一句中学物理——汇到同一根导线上的电流会自动相加。CIM 借这条定律，让一整列 cell 的电流在竖线上自己加起来，不用任何指令，就完成了“求和”。
die（裸片）：从一整片晶圆上切下来的一小块芯片，就是“一颗芯片的本体”。面积常用 mm²（平方毫米）算，比如 280/380/450 mm²。
chiplet（芯粒）：把一颗大芯片拆成几块小的来造，每块叫一个 chiplet。小块良率高、更便宜，再拼到一起用。AMD/苹果的大芯片都这么干。
interposer（转接板）：2.5D 封装里那块“底板”——几个 chiplet 并排坐在它上面，它负责把彼此用极密的线连通。像把几块积木粘到同一块底座上。
TSV = 硅通孔（Through-Silicon Via）：在芯片上垂直打的微小通孔，让上下叠起来的几层芯片能直接连通。3D 堆叠就靠它上下传信号。
2.5D vs 3D：都是“拼很多芯片”的封装。2.5D=几块芯片并排坐在转接板上（平铺）；3D=几层芯片垂直叠起来用 TSV 上下连。3D 同样面积密度最高，但热量难散、良率更难，所以“大面积 3D”至今很少有人做。
yield（良率）：造出来的芯片里“合格能用”的比例。晶圆上有随机缺陷，芯片面积越大、单片里 macro 越多，整片全好的概率越低，良率越低 → 成本越高。这是“大芯片”的天然代价。
冗余修复（redundancy）：多造一些备用的行/列/macro，造好后用好的去顶替坏的。能把一部分本来报废的芯片救回来，提升良率。
transformer 层：现代 LLM 的基本积木，一摞同样结构的层叠起来。每层主要就是几个大权重矩阵（注意力 + 前馈）。一个字从下往上穿过每一层，每层做一次大矩阵乘（=一堆 MAC）。层数越多、矩阵越大，模型越强也越大。
