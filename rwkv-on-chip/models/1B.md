# ~1B 设计点 × 我们的三值 CIM 芯片

*设计点参数为圆整估算；内存为算术估算；芯片容量是设计目标，尚未流片。*

**判决：1GB 单片 ✅（宽裕——最省电的常开地板档）**

## 参数拆解
- 设计点 C = 2048，L = 15（圆整设计点，非特定 checkpoint）
- ~1.0B 参数

## 三值后内存（见 [`../sizing.py`](../sizing.py)）
| 项 | 大小 |
|---|---|
| 三值权重（transformer，1.58 bit） | 149 MB |
| emb + head（int8，不可三值化） | 268 MB |
| 递归状态（int8，batch=1） | ~1 MB |
| **片上合计** | **~0.42 GB** |

## 上不上片
- **1GB 片**：✅ 宽裕（占~42%，留一半余量给更大上下文 / 多模态编码器）
- **3GB 片**：✅
- 单片 / 多片：**单片**

## 适用场景
最省电的常开档：录音笔、可穿戴、耳机、玩具——只要"够用的智能 + 全天电池 + 断网私有"。注意：emb+head（大词表税）反而成了主要占用，留给芯片设计的提示是——小模型时，**词表压缩 / 共享 emb-head** 是省片上内存的下一个杠杆。
