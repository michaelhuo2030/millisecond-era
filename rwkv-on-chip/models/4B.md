# ~4B 设计点 × 我们的三值 CIM 芯片

*设计点参数为圆整估算；内存为算术估算；芯片容量是设计目标，尚未流片。*

**判决：1GB ❌ 临界（~1.12GB，正好压线）· 3GB 单片 ✅**

## 参数拆解
- 设计点 C = 3072，L = 32（圆整设计点）
- ~4.0B 参数

## 三值后内存（见 [`../sizing.py`](https://github.com/michaelhuo2030/millisecond-era/blob/main/rwkv-on-chip/sizing.py)）
| 项 | 大小 |
|---|---|
| 三值权重（transformer，1.58 bit） | 716 MB |
| emb + head（int8，不可三值化） | 403 MB |
| 递归状态（int8，batch=1） | ~5 MB |
| **片上合计** | **~1.12 GB** |

## 上不上片
- **1GB 片**：❌ **正好压在线上**（~1.12GB，超 1GB 约 12%）——诚实讲，4B 不是 1GB 这颗的舒适区
- **3GB 片**：✅ 单片，宽裕
- 单片 / 多片：3GB 片单片；1GB 片需轻度切分或更激进的 emb 压缩

## 适用场景
中端边缘 / 机器人 / 平衡点——能力与功耗之间的甜点。但它揭示了一条诚实边界：**4B 是 1GB 与 3GB 两档的分水岭**。要单片吃下 4B，就该把密度目标推到 3GB。
