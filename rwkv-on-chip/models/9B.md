# ~9B 设计点 × 我们的三值 CIM 芯片

`config [ESTIMATE]` · `sizing [ESTIMATE]` · `chip [TARGET]`

**判决：1GB ❌ · 3GB 单片 ✅（~2.21GB——3GB 目标片的旗舰单片档）**

## 参数拆解
- 设计点 C = 4096，L = 42（圆整设计点 `[ESTIMATE]`）
- ~9.0B 参数

## 三值后内存 `[ESTIMATE]`（见 [`../sizing.py`](../sizing.py)）
| 项 | 大小 |
|---|---|
| 三值权重（transformer，1.58 bit） | 1670 MB |
| emb + head（int8，不可三值化） | 537 MB |
| 递归状态（int8，batch=1） | ~11 MB |
| **片上合计** | **~2.21 GB** |

## 上不上片
- 1GB `[TARGET]`：❌（光三值权重就 1.67GB）
- 3GB `[TARGET]`：✅ **单片**（占~74%，留~26% 余量）
- 单片 / 多片：**3GB 目标片单片**

## 适用场景
> **这一档是关键的设计决策点。** 9B 上不了 1GB 片——但只要我们把片上密度目标从 1GB 推到 **3GB**（这是 ReRAM/CIM 密度可达的设计余量），**9B 就单片吃得下（~2.21GB < 3GB）**。

也就是说：芯片容量不是写死的 1GB，而是一个可滑动的设计旋钮。1GB 这颗是 1–3B 边缘单流的命；3GB 这颗把整个 7–9B 档都纳入单片私有 reasoning。高端私有助手 / 离线工作站就落在这里。
